## 1. Основы PHP и возможности языка

## Числа с плавающей точкой (float)
- https://www.php.net/manual/en/language.types.float.php

### Почему числа с плавающей точкой часто неточны в PHP?
**Ответ:** Числа с плавающей точкой имеют ограниченную точность и хранятся в двоичном формате (IEEE 754). Это означает, что некоторые десятичные дроби (например, 0.1 или 0.2) не могут быть представлены точно.
[Подробное объяснение чисел с плавающей точкой](answers/floating_point_numbers.md)

### Junior
#### Что означает аббревиатура PHP и каково его основное назначение?
**Ответ:** Изначально PHP расшифровывалось как "Personal Home Page", но сейчас это рекурсивный акроним "PHP: Hypertext Preprocessor". Это серверный скриптовый язык, предназначенный для веб-разработки. Его основное назначение — генерация динамического веб-контента, обработка данных форм, взаимодействие с базами данных, управление сессиями и многое другое.

#### Является ли PHP регистрозависимым языком?
**Ответ:** Да, PHP регистрозависим для имён переменных. Однако имена функций, имена классов и встроенные конструкции (такие как `echo`, `if` и т. д.) регистронезависимы.

#### Каковы правила именования переменных в PHP?
**Ответ:**
- Имя должно начинаться со знака `$`.
- За ним следует буква или символ подчёркивания.
- Может содержать только буквы, цифры и символы подчёркивания (A-z, 0-9 и _).
- Не может начинаться с цифры.
- Имена переменных регистрозависимы.

#### Какие типы данных существуют в PHP?
**Ответ:**
- **Скалярные:** `int`, `float`, `string`, `bool`.
- **Составные:** `array`, `object`, `callable`, `iterable`.
- **Специальные:** `resource`, `null`.
[Подробное руководство по типам данных](answers/data_types.md)

#### В чём разница между `empty()`, `is_null()` и `isset()`?
**Ответ:**
- `empty()`: Возвращает true, если переменная является пустой строкой, 0, false, null или пустым массивом.
- `is_null()`: Возвращает true только если переменная равна null.
- `isset()`: Возвращает true, если переменная определена и не равна null.

#### В чём разница между `echo` и `print`?
**Ответ:**
- `echo` — это языковая конструкция, а `print` — функция (хотя ведёт себя как конструкция).
- `echo` немного быстрее и может принимать несколько параметров.
- `print` всегда возвращает 1, поэтому может использоваться в выражениях.

#### Что такое `include()`, `require()`, `include_once()` и `require_once()`?
**Ответ:**
- `include()`: Подключает файл. В случае неудачи выдаёт предупреждение и продолжает выполнение.
- `require()`: Подключает файл. В случае неудачи выдаёт фатальную ошибку и останавливает выполнение.
- `*_once()`: Аналогичны своим парам, но гарантируют, что файл будет подключён только один раз.

#### Какие типы массивов существуют в PHP?
**Ответ:**
- **Индексированные массивы:** Массивы с числовым индексом.
- **Ассоциативные массивы:** Массивы с именованными ключами.
- **Многомерные массивы:** Массивы, содержащие один или несколько массивов.

#### В чём разница между строками в одинарных и двойных кавычках?
**Ответ:**
- **Одинарные кавычки:** Литералы. Переменные и большинство escape-последовательностей не интерпретируются.
- **Двойные кавычки:** Переменные подставляются, а escape-последовательности (такие как `\n`, `\t`) интерпретируются.

#### В чём разница между `$message` и `$$message`?
**Ответ:** `$message` — это обычная переменная. `$$message` — это «переменная переменной», которая использует значение `$message` в качестве имени другой переменной.

#### Что подразумевается под «переходом в PHP»?
**Ответ:** Это означает встраивание PHP-кода в HTML с помощью тегов `<?php ... ?>`.

#### Каковы правила определения «истинности» любого значения (truthiness)?
**Ответ:** В PHP значения считаются «истинными», если они не являются: `null`, `false`, `0`, `0.0`, `""` (пустая строка), `"0"` или `[]` (пустой массив).

#### В чём разница между константами и переменными?
**Ответ:** Константы определяются с помощью `define()` или `const` и не могут быть изменены или удалены после определения. Переменные могут менять своё значение в любой момент.

#### Назовите некоторые встроенные константы PHP.
**Ответ:** `PHP_VERSION`, `PHP_OS`, `DEFAULT_INCLUDE_PATH`, `DIRECTORY_SEPARATOR`, `E_ERROR`, `E_WARNING`.

#### Для чего используются `explode()` и `implode()`?
**Ответ:**
- `explode()`: Разбивает строку в массив по разделителю.
- `implode()`: Объединяет элементы массива в строку с помощью разделителя.

#### В чём разница между `unset()` и `unlink()`?
**Ответ:**
- `unset()`: Уничтожает переменную или элемент массива.
- `unlink()`: Удаляет файл с сервера.

### Middle
#### Что такое магические константы в PHP?
**Ответ:** Магические константы — это предопределённые константы, значения которых зависят от того, где они используются.
- `__LINE__`, `__FILE__`, `__DIR__`, `__FUNCTION__`, `__CLASS__`, `__TRAIT__`, `__METHOD__`, `__NAMESPACE__`, `ClassName::class`.

#### Что такое генераторы?
**Ответ:** Генераторы предоставляют простой способ реализации итераторов с помощью ключевого слова `yield`, позволяя перебирать данные без создания массива в памяти.
[Генераторы и yield](answers/generators.md)

#### В чём разница между замыканием и анонимной функцией?
**Ответ:** Анонимная функция — это функция без имени. Замыкание (Closure) — это анонимная функция, которая может «захватывать» переменные из родительской области видимости с помощью ключевого слова `use`.

#### Что такое Type Hinting?
**Ответ:** Type Hinting позволяет указывать ожидаемый тип данных для аргументов функции и возвращаемых значений. Это помогает выявлять ошибки и улучшает читаемость кода.

#### Что такое Garbage Collection в PHP?
**Ответ:**
PHP использует механизм подсчёта ссылок и циклический сборщик мусора для автоматического освобождения памяти, занятой объектами и переменными, которые больше недоступны.

- **Подсчёт ссылок:** В PHP 7+ `refcount` был перенесён из самой структуры `zval` внутрь структур сложных значений (`zend_string`, `zend_array` и др.). Простые типы (например, `int` или `bool`) не используют подсчёт ссылок. Когда `refcount` достигает нуля, память освобождается.
- **Что такое «цикл»?** Цикл возникает, когда объекты ссылаются друг на друга (циклические ссылки), что не позволяет их `refcount` достичь нуля даже после удаления.
- **Как это очищается:** Когда «корневой буфер» достигает предела, сборщик временно уменьшает внутренние refcount. Если refcount достигает нуля, объект помечается как мусор и удаляется.
- **Производительность:** GC предотвращает утечки памяти (критически важно для демонов/воркеров), но добавляет нагрузку на CPU во время циклов сборки.

[Подробное руководство по Garbage Collection](answers/garbage_collector.md)

#### Что такое zval и какова его базовая структура? ⭐ **Важно**
**Ответ:** `zval` (Zend value) — это фундаментальная структура данных, используемая Zend Engine для представления любой переменной PHP. В PHP 7+ она была значительно оптимизирована для снижения потребления памяти (до 16 байт) и отказа от лишних аллокаций в куче.

**Внутренняя структура (PHP 7+):**
1. **value**: 8-байтовое объединение (union), хранящее фактические данные (например, `lval` для целых чисел, `dval` для чисел с плавающей точкой или указатели, такие как `*str`, `*arr`, `*obj`).
2. **u1 (type_info)**: 4-байтовое объединение, содержащее **тег типа** (например, `IS_STRING`, `IS_LONG`) и **флаги типа** (указывающие, является ли значение ссылочно-зависимым или участвует ли оно в сборке мусора).
3. **u2**: 4-байтовое объединение, используемое для вспомогательных данных (например, цепочки коллизий в хеш-таблицах массивов).

**Основные отличия от PHP 5:**
- **Размер**: Уменьшен с 24/32 байт до 16 байт.
- **Подсчёт ссылок**: Перенесён из самого `zval` внутрь структур сложных значений (`zend_string`, `zend_array` и т.д.).
- **Простые типы**: Такие типы, как `null`, `bool`, `int` и `float`, хранятся непосредственно в `zval` и больше не требуют подсчёта ссылок или выделения памяти в куче.

[Подробное руководство по структуре Zval](https://www.phpinternalsbook.com/php7/zvals/basic_structure.html)


#### Что такое copy-on-write в PHP и когда передача по значению дублирует данные? ⭐ **Важно**
**Ответ:** **Copy-on-write (CoW, копирование при записи)** — это оптимизация, при которой PHP разделяет один `zval` между несколькими переменными до тех пор, пока одна из них не будет изменена.
- **Только чтение**: передача большого массива по значению лишь создает ссылку на тот же `zval` (память не дублируется).
- **Модификация**: изменение массива вызывает «отщепление» (separation), создавая реальную копию в памяти.
- **Объекты**: передаются по **хэндлу**, поэтому изменение свойств не вызывает дублирования всей структуры объекта, как у массивов.
Не используйте `&` только ради экономии памяти; CoW уже эффективно справляется с этой задачей при чтении.
[Подробное руководство по Copy-on-Write](answers/copy_on_write.md)

#### Что такое OPCache?
**Ответ:** OPCache — это механизм кеширования для PHP, который хранит предварительно скомпилированный байт-код скриптов в разделяемой памяти, устраняя необходимость загрузки и парсинга скриптов PHP при каждом запросе.

#### Как работает обработка ошибок в PHP?
**Ответ:** PHP использует несколько механизмов:
- **Уровни ошибок:** Настраиваются через `error_reporting()` и `display_errors` в `php.ini`.
- **Блоки Try-Catch:** Для обработки `Exceptions` и `Errors` (начиная с PHP 7).
- **Пользовательские обработчики ошибок:** С помощью `set_error_handler()` и `set_exception_handler()`.

#### Каковы риски использования PHP как долгоживущего процесса (демона)?
**Ответ:** Утечки памяти и загрязнение глобального состояния — главные риски, поскольку переменные и объекты сохраняются между запросами.
[Запуск PHP как демона](answers/php_advanced_extras/#php-as-a-daemon-swoole-roadrunner)

#### Как работает OPCache?
**Ответ:** OPCache повышает производительность, сохраняя предварительно скомпилированный байт-код в разделяемой памяти, что позволяет избежать загрузки, парсинга и компиляции скриптов PHP при каждом запросе.

#### Как работает OPCache начиная с версии 8.0 и новый JIT?
**Ответ:** OPCache с JIT (начиная с 8.0) определяет «горячий» код и компилирует его в нативный машинный код, позволяя процессору выполнять его напрямую, минуя цикл интерпретатора Zend VM.

#### Что вы знаете о JIT?
**Ответ:** JIT (Just-In-Time) высокоэффективен для CPU-интенсивных задач, но обычно даёт минимальные преимущества для стандартных I/O-зависимых веб-приложений. Он позволяет использовать PHP не только для веб-разработки (например, для машинного обучения, обработки изображений).
[Подробное руководство по OPCache и JIT](answers/opcache_jit.md)

#### Почему `var_dump(0.1 + 0.2 == 0.3)` возвращает `bool(false)`?
**Ответ:**
Это происходит потому, что числа с плавающей точкой в PHP (и большинстве других языков) следуют стандарту **IEEE 754**, который представляет их в двоичном формате. Многие десятичные дроби, такие как `0.1` и `0.2`, не могут быть точно представлены как конечные двоичные дроби.
- При сложении `0.1 + 0.2` внутренний результат немного отличается от точного `0.3` (приблизительно `0.30000000000000004`).
- **Сравнение:** Никогда не используйте `==` для сравнения float. Вместо этого используйте малый epsilon: `abs(($a + $b) - $c) < 0.00001`.

**Точность в базах данных и финансах:**
- В базах данных такие значения необходимо хранить как **`DECIMAL`** (или `NUMERIC`), а не как `float` или `double`.
- **Причина:** `DECIMAL` хранит числа как значения с фиксированной точкой (или внутренне как строки), обеспечивая точность вычислений, что критично для денежных значений. Типы с плавающей точкой предназначены для приблизительных научных значений, где скорость важнее абсолютной точности.

## 2. Объектно-ориентированное программирование (ООП)

### Junior
#### Что такое класс и объект?
**Ответ:**
- **Класс:** Шаблон или чертёж для создания объектов. Он определяет свойства (атрибуты) и методы (поведение).
- **Объект:** Экземпляр класса. Конкретная реализация шаблона со своим собственным состоянием.

#### Каковы основные характеристики ООП?
**Ответ:**
- **Инкапсуляция:** Объединение данных и методов, работающих с этими данными, в единый блок (класс) с ограничением прямого доступа.
- **Наследование:** Механизм, при котором подкласс наследует свойства и поведение суперкласса.
- **Полиморфизм:** Способность различных классов обрабатываться как экземпляры одного интерфейса или родительского класса.
- **Абстракция:** Сокрытие сложных деталей реализации и предоставление только существенных характеристик объекта.

#### Какие модификаторы доступа существуют в PHP?
**Ответ:**
- `public`: Доступен отовсюду.
- `protected`: Доступен внутри самого класса и наследующих классов.
- `private`: Доступен только внутри класса, в котором определён.

#### Что такое конструкторы и деструкторы?
**Ответ:**
- `__construct()`: Специальный метод, вызываемый автоматически при создании объекта.
- `__destruct()`: Специальный метод, вызываемый при уничтожении объекта или завершении скрипта.

### Middle
#### В чём разница между `this`, `self` и `static`?
**Ответ:**
- `$this`: Ссылается на текущий экземпляр объекта.
- `self`: Ссылается на текущий класс. Используется для статических свойств/методов.
- `static`: Ссылается на вызываемый класс (Late Static Binding).
[this vs self vs static](answers/this_self_static.md)

#### В чём разница между Interface и Abstract Class?
**Ответ:**
- **Interface:** Определяет контракт. Может содержать только публичные методы без реализации. Класс может реализовывать несколько интерфейсов.
- **Abstract Class:** Может содержать как абстрактные методы, так и конкретные методы с реализацией. Класс может наследоваться только от одного абстрактного класса.
[Сравнение Interface и Abstract Class](answers/interface_vs_abstract.md)

#### Что такое Traits?
**Ответ:** Traits — это механизм повторного использования кода в языках с одиночным наследованием, таких как PHP. Они позволяют включать наборы методов в несколько независимых классов.

#### Что такое переопределение методов (Method Overriding)?
**Ответ:** Переопределение методов происходит, когда подкласс предоставляет собственную реализацию метода, уже определённого в его родительском классе.

#### Какие типы наследования существуют?
**Ответ:**
- **Одиночное:** Класс наследуется от одного родительского класса.
- **Многоуровневое:** Класс наследуется от родителя, который сам является подклассом.
- **Иерархическое:** Несколько подклассов наследуются от одного родительского класса.
- **Гибридное:** Комбинация двух или более вышеперечисленных (с использованием Traits/Interfaces).
*Примечание: PHP не поддерживает множественное наследование напрямую.*

#### Что такое Namespaces в PHP?
**Ответ:** Пространства имён предоставляют способ группировки связанных классов, функций и констант для предотвращения конфликтов имён.

#### В чём разница между абстракцией и инкапсуляцией?
**Ответ:**
- **Абстракция:** Сокрытие сложных деталей реализации и демонстрация только существенных характеристик объекта (фокус на «Что»).
- **Инкапсуляция:** Объединение данных и методов вместе с ограничением прямого доступа к некоторым компонентам объекта (фокус на «Как»).

#### Что такое магические методы?
**Ответ:** Специальные методы, начинающиеся с `__`, которые вызываются при определённых событиях (например, `__get`, `__set`, `__call`, `__toString`, `__sleep`, `__wakeup`).


### Senior
#### Что такое перегрузка методов (Method Overloading) в PHP?
**Ответ:** PHP не поддерживает традиционную перегрузку методов (одинаковое имя, разные аргументы). Вместо этого используются магические методы, такие как `__call()`, для её имитации.
[Перегрузка методов в PHP](answers/method_overloading.md)

## 3. Архитектура ПО и принципы

### Middle
#### Что такое принципы SOLID?
**Ответ:**
- **S:** Single Responsibility (Принцип единственной ответственности)
- **O:** Open/Closed (Принцип открытости/закрытости)
- **L:** Liskov Substitution (Принцип подстановки Лисков)
- **I:** Interface Segregation (Принцип разделения интерфейса)
- **D:** Dependency Inversion (Принцип инверсии зависимостей)
[Руководство по принципам SOLID](answers/solid_principles.md)

#### Что такое CQRS?
**Ответ:** Command Query Responsibility Segregation. Паттерн, разделяющий чтение данных (Query) и запись/обновление данных (Command).

#### В чём разница между Entity и Value Object в DDD?
**Ответ:** 
- **Entity (Сущность):** Объект, обладающий уникальным идентификатором (ID). Сущность остается собой, даже если её свойства меняются со временем (например, пользователь остается тем же пользователем, если сменит email).
- **Value Object (Объект-значение):** Объект, определяемый только совокупностью своих атрибутов (без ID). Они иммутабельны (неизменяемы). Два объекта-значения с одинаковыми свойствами считаются эквивалентными (например, Money, Color, Address).

#### В чём ключевое различие между гексагональной (Hexagonal) и луковой (Onion) архитектурами?
**Ответ:** 
- **Hexagonal (Ports & Adapters):** Делает акцент на границе между приложением и внешним миром через порты (интерфейсы) и адаптеры (реализации), не навязывая строгую структуру внутри «гексагона».
- **Onion Architecture:** Предписывает строгое деление на концентрические слои. Зависимости всегда направлены внутрь: от инфраструктуры к ядру (Domain Model), которое находится в самом центре.
[Подробный разбор архитектур](answers/architecture_advanced.md) | [Практические примеры](answers/architecture_examples.md)

### Senior
#### Что такое Anemic и Rich модели?
**Ответ:**
- **Anemic Model:** Модели, содержащие только данные (геттеры/сеттеры), но не бизнес-логику.
- **Rich Model:** Модели, инкапсулирующие и данные, и бизнес-логику (Domain-Driven Design).
[Подробный разбор Anemic vs Rich Model](answers/architecture_advanced.md)

#### Что такое GRASP?
**Ответ:** General Responsibility Assignment Software Patterns. Набор принципов назначения ответственностей классам и объектам.
[Руководство по паттернам GRASP](answers/architecture_advanced.md)

#### Каковы плюсы и минусы микросервисов?
**Ответ:**
- **Плюсы:** Независимое масштабирование, скорость и гибкость, технологическая независимость и изоляция сбоев.
- **Минусы:** Сложность управления, проблемы с согласованностью данных (распределённые транзакции), сетевая задержка и сложный мониторинг.
[Микросервисная архитектура](answers/architecture_highload.md)

## 4. Принципы проектирования систем (Middle+)

### Middle

#### Что такое проектирование систем и почему это важно?
**Ответ:** Проектирование систем (System Design) — это процесс определения архитектуры, компонентов, модулей, интерфейсов и данных системы для удовлетворения заданных требований. Это крайне важно, потому что хорошо спроектированная система может эффективно масштабироваться, корректно обрабатывать сбои и обслуживаться с минимальными затратами. Плохое проектирование может привести к огромным финансовым потерям и системным отказам.

#### В чём разница между функциональными и нефункциональными требованиями?
**Ответ:**
- **Функциональные требования:** Определяют, *что* должна делать система. Это конкретные функции, например «пользователь может забронировать самокат» или «пользователь может посмотреть историю поездок».
- **Нефункциональные требования:** Определяют, *как* система должна работать. Они включают такие качества, как масштабируемость (обработка 100 тыс. запросов), доступность (99.9% uptime), надёжность и безопасность.

#### Объясните вертикальное и горизонтальное масштабирование.
**Ответ:**
- **Вертикальное масштабирование (Scaling Up):** Увеличение мощности одного сервера (например, добавление CPU, RAM или более быстрых SSD). Это просто, но ограничено физическими возможностями одной машины и становится очень дорогим на верхнем уровне.
- **Горизонтальное масштабирование (Scaling Out):** Добавление новых серверов в пул и распределение нагрузки между ними с помощью Load Balancer. Это позволяет практически неограниченный рост и обеспечивает лучшую отказоустойчивость, так как система может продолжать работу, если один узел выходит из строя.

#### Каковы основные стратегии масштабирования базы данных?
**Ответ:**
- **Репликация:** Создание копий базы данных (Master/Slave). Обычно Master обрабатывает запись, а Slave обрабатывают запросы на чтение для распределения нагрузки.
- **Шардирование:** Распределение данных по нескольким независимым серверам баз данных (шардам) на основе определённого ключа (например, User ID, геолокация). Каждый шард содержит подмножество общих данных.
- **Партиционирование:** Разделение большой таблицы на более мелкие, управляемые логические части внутри одного экземпляра базы данных для улучшения производительности запросов.

#### Что такое Sharding Key и как выбрать хороший?
**Ответ:** Sharding Key — это атрибут, используемый для определения, к какому шарду относится конкретная часть данных. Хороший Sharding Key обеспечивает:
- **Равномерное распределение:** Данные распределяются равномерно по шардам, избегая «горячих точек».
- **Эффективность запросов:** Минимизирует необходимость «межшардовых» запросов, сохраняя связанные данные вместе.
Распространённые ключи включают User ID, Tenant ID или хеш уникального атрибута.

#### Каковы лучшие практики проектирования REST API?
**Ответ:** Используйте существительные во множественном числе для ресурсов (например, /users), стандартные HTTP-методы (GET, POST, PUT, DELETE), версионирование (например, /v1/...) и правильные HTTP-коды состояния для обозначения результата запроса.

#### Какова связь между Latency, Throughput и Concurrency?
**Ответ:** Они связаны **законом Литтла**: `Concurrency = Throughput × Latency`.
- **Latency:** Время обработки одного запроса.
- **Throughput:** Количество запросов, которое система может обработать в единицу времени.
- **Concurrency:** Количество запросов, обрабатываемых одновременно.
Если вы увеличиваете количество параллельных запросов (concurrency) без увеличения пропускной способности системы (throughput), задержка (latency) автоматически возрастёт, что приведёт к узкому месту.

#### Что такое Load Balancer и каковы распространённые алгоритмы балансировки?
**Ответ:** Load Balancer (например, Nginx, HAProxy, AWS ELB) — это компонент, распределяющий входящий сетевой трафик по нескольким серверам, чтобы ни один сервер не был перегружен.
Распространённые алгоритмы:
- **Round Robin:** Запросы распределяются последовательно.
- **Least Connections:** Трафик направляется на сервер с наименьшим количеством активных соединений.
- **IP Hash:** IP-адрес клиента определяет, какой сервер получит запрос (полезно для сохранения сессий).
- **Latency-Aware:** Балансировщик отслеживает историю времени отклика и избегает «медленных» узлов.

#### Что такое паттерны Circuit Breaker и Back Pressure?
**Ответ:**
- **Circuit Breaker:** Отслеживает вызовы к отказывающему сервису. Если частота ошибок превышает порог, «цепь размыкается», и система прекращает вызовы к сервису, возвращая быстрый fallback-ответ вместо ожидания таймаутов. Это предотвращает каскадные отказы.
- **Back Pressure:** Механизм, при котором система «отталкивает» высокий поток входящих запросов, которые она не может обработать. Вместо того чтобы позволять очередям расти бесконечно (что увеличивает latency), система отклоняет новые запросы (например, HTTP 429) или просит клиента замедлиться.

#### Что такое Idempotency в проектировании API и почему это важно?
**Ответ:** Идемпотентная операция — это операция, которая даёт одинаковый эффект независимо от того, сколько раз она выполнена. Например, `PUT` и `DELETE` должны быть идемпотентными, тогда как `POST` обычно нет.
В распределённых системах идемпотентность критически важна для обработки сетевых сбоев. Если клиент не получил подтверждение из-за таймаута, он может повторить запрос. Если API идемпотентно, повторная попытка не вызовет нежелательных побочных эффектов (например, двойного списания с клиента).

### Senior

#### Как можно снизить Tail Latency в распределённой системе?
**Ответ:** Tail Latency (P99+) можно снизить с помощью нескольких техник:
- **Hedged Requests (Tied Requests):** Отправка одного и того же запроса нескольким репликам и использование результата от первой ответившей.
- **Request Coalescing:** Объединение идентичных параллельных запросов в один вызов к бэкенду.
- **Background Jittering:** Добавление случайных задержек к запланированным задачам для предотвращения проблемы «thundering herd».
- **Deadline Propagation:** Передача максимально допустимого времени для запроса всем подсервисам, чтобы они могли прерваться досрочно, если дедлайн уже истёк.
- **Latency-Aware Balancing:** Избегание узлов, работающих в данный момент с низкой производительностью.

#### Сравните различные стратегии маршрутизации шардов: Application-level, Proxy и Coordinator.
**Ответ:**
- **Application-Level Routing:** Логика определения нужного шарда содержится в коде приложения. Это быстро (нет дополнительного сетевого хопа), но сложно поддерживать, так как логика шардирования дублируется между сервисами.
- **Proxy-Based Routing:** Отдельный слой (например, Vitess, ProxySQL) располагается между приложением и базой данных. Он обрабатывает маршрутизацию, делая шардирование прозрачным для приложения. Добавляет небольшой сетевой хоп, но централизует управление.
- **Coordinator-Based:** Отдельный сервис (например, Zookeeper или пользовательский metadata-сервис) отслеживает расположение шардов. Клиенты запрашивают координатор, чтобы найти нужный шард. Это надёжно, но сложно в реализации.

#### Сравните 2PC, 3PC, TCC и паттерн Saga.
**Ответ:**
- **2PC (Two-Phase Commit):** Синхронный протокол для распределённых транзакций. Обеспечивает строгую согласованность, но является блокирующим и может стать узким местом производительности.
- **3PC (Three-Phase Commit):** Расширение 2PC, добавляющее фазу "Pre-commit" для уменьшения блокировок в случае отказа координатора. Редко используется в продакшене.
- **TCC (Try-Confirm-Cancel):** Паттерн на уровне приложения. Вы резервируете ресурсы (Try), а затем либо подтверждаете (Confirm), либо освобождаете (Cancel). Гибкий, но сложный в реализации.
- **Saga Pattern:** Последовательность локальных транзакций, где каждый шаг имеет соответствующую «компенсирующую транзакцию» для отмены изменений в случае сбоя. Высоко масштабируем и распространён в микросервисах, но обеспечивает только eventual consistency.

#### Объясните разницу между Eventual и Strong Consistency.
**Ответ:**
- **Eventual Consistency:** Реплики в конечном итоге придут к одному значению, но возможна задержка, при которой разные пользователи видят разные данные. Обеспечивает высокую доступность и низкую задержку. Подходит для комментариев, лайков или каталогов товаров.
- **Strong Consistency:** Гарантирует, что после успешной записи любое последующее чтение вернёт новое значение. Необходима для финансовых транзакций, управления складом или систем бронирования.
- **Linearizability:** Более строгая форма согласованности, при которой операции выглядят так, будто происходят мгновенно и в том порядке, в котором были отправлены.

#### Что такое Tail Latency и почему это важно в распределённых системах?
**Ответ:** Tail Latency — это высокая задержка, которую испытывает небольшой процент запросов (например, 99.9-й перцентиль). В крупномасштабной системе, где один пользовательский запрос может вызвать сотни подзапросов к различным микросервисам, высокая tail latency даже в одном сервисе может замедлить весь ответ для пользователя, что делает её критической метрикой надёжности системы.
[Подробный разбор принципов проектирования систем](answers/system_design_principles.md)

## 5. Паттерны проектирования
Паттерны проектирования тесно связаны с разделом [Чистый код и лучшие практики](#16-чистый-код-и-лучшие-практики).

### Junior
#### Каковы основные категории паттернов проектирования?
**Ответ:** Паттерны проектирования обычно разделяются на три группы:
- **Порождающие паттерны (Creational):** Фокусируются на механизмах создания объектов.
- **Структурные паттерны (Structural):** Фокусируются на том, как классы и объекты компонуются в более крупные структуры.
- **Поведенческие паттерны (Behavioral):** Фокусируются на взаимодействии между объектами.

#### Что такое паттерн Singleton?
**Ответ:** Паттерн, гарантирующий, что класс имеет только один экземпляр, и предоставляющий глобальную точку доступа к нему. Относится к порождающим паттернам.

### Middle
#### Что такое паттерн Factory Method?
**Ответ:** Factory Method — это порождающий паттерн проектирования, который предоставляет интерфейс для создания объектов в суперклассе, но позволяет подклассам изменять тип создаваемых объектов.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы избегаете жёсткой привязки между создателем и конкретными продуктами. Принцип единственной ответственности. Вы можете вынести код создания продукта в одно место программы, упрощая поддержку. Принцип открытости/закрытости. Вы можете добавлять новые типы продуктов в программу, не ломая существующий клиентский код.
- **Минусы:** Код может стать сложнее, поскольку для реализации паттерна нужно вводить множество новых подклассов. Лучший сценарий — когда вы вводите паттерн в уже существующую иерархию классов-создателей.

#### Что такое паттерн Abstract Factory?
**Ответ:** Abstract Factory — это порождающий паттерн проектирования, позволяющий создавать семейства связанных объектов без указания их конкретных классов.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете быть уверены, что продукты, создаваемые фабрикой, совместимы друг с другом. Вы избегаете жёсткой привязки между конкретными продуктами и клиентским кодом. Принцип единственной ответственности. Вы можете вынести код создания продуктов в одно место, упрощая поддержку. Принцип открытости/закрытости. Вы можете вводить новые варианты продуктов, не ломая существующий клиентский код.
- **Минусы:** Код может стать сложнее, чем нужно, поскольку вместе с паттерном вводится множество новых интерфейсов и классов.

#### Что такое паттерн Builder?
**Ответ:** Builder — это порождающий паттерн проектирования, позволяющий конструировать сложные объекты пошагово. Паттерн позволяет создавать различные типы и представления объекта, используя один и тот же код конструирования.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете конструировать объекты пошагово, откладывать шаги или выполнять их рекурсивно. Вы можете переиспользовать один и тот же код конструирования при создании различных представлений продуктов. Принцип единственной ответственности. Вы можете отделить сложный код конструирования от бизнес-логики продукта.
- **Минусы:** Общая сложность кода возрастает, поскольку паттерн требует создания нескольких новых классов.

#### Что такое паттерн Prototype?
**Ответ:** Prototype — это порождающий паттерн проектирования, позволяющий копировать существующие объекты без привязки к их конкретным классам.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете клонировать объекты без привязки к их конкретным классам. Вы можете избавиться от повторяющегося кода инициализации в пользу клонирования заранее настроенных прототипов. Вы можете создавать сложные объекты удобнее. Вы получаете альтернативу наследованию при работе с конфигурационными пресетами для сложных объектов.
- **Минусы:** Клонирование сложных объектов с циклическими ссылками может быть весьма непростым.

#### Что такое паттерн Adapter?
**Ответ:** Adapter — это структурный паттерн проектирования, позволяющий объектам с несовместимыми интерфейсами работать вместе.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип единственной ответственности. Вы можете отделить код конвертации интерфейсов или данных от основной бизнес-логики программы. Принцип открытости/закрытости. Вы можете вводить новые типы адаптеров в программу, не ломая существующий клиентский код, при условии, что они работают с адаптерами через клиентский интерфейс.
- **Минусы:** Общая сложность кода возрастает, так как нужно вводить набор новых интерфейсов и классов. Иногда проще просто изменить класс сервиса так, чтобы он соответствовал остальному коду.

#### Что такое паттерн Bridge?
**Ответ:** Bridge — это структурный паттерн проектирования, позволяющий разделить большой класс или набор тесно связанных классов на две отдельные иерархии — абстракцию и реализацию, которые могут развиваться независимо друг от друга.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете создавать платформонезависимые классы и приложения. Клиентский код работает с абстракциями высокого уровня, не имея дела с деталями платформы. Принцип открытости/закрытости. Вы можете вводить новые абстракции и реализации независимо друг от друга. Принцип единственной ответственности. Вы можете сосредоточиться на высокоуровневой логике в абстракции и на деталях платформы в реализации.
- **Минусы:** Вы можете усложнить код, применяя паттерн к и без того связному классу.

#### Что такое паттерн Composite?
**Ответ:** Composite — это структурный паттерн проектирования, позволяющий компоновать объекты в древовидные структуры и затем работать с этими структурами так, как если бы они были отдельными объектами.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете удобнее работать со сложными древовидными структурами, используя полиморфизм и рекурсию. Принцип открытости/закрытости. Вы можете вводить новые типы элементов в приложение, не ломая существующий код, работающий с деревом объектов.
- **Минусы:** Может быть сложно предоставить общий интерфейс для классов, функциональность которых сильно различается. В некоторых сценариях придётся чрезмерно обобщать интерфейс компонента, что затрудняет его понимание.

#### Что такое паттерн Decorator?
**Ответ:** Decorator — это структурный паттерн проектирования, позволяющий добавлять новое поведение объектам, помещая их внутрь специальных объектов-обёрток, содержащих это поведение.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете расширять поведение объекта без создания нового подкласса. Вы можете добавлять или удалять обязанности объекта во время выполнения. Вы можете комбинировать несколько поведений, оборачивая объект в несколько декораторов. Принцип единственной ответственности. Вы можете разделить монолитный класс, реализующий множество вариантов поведения, на несколько более мелких классов.
- **Минусы:** Сложно удалить конкретную обёртку из стека обёрток. Сложно реализовать декоратор так, чтобы его поведение не зависело от порядка в стеке декораторов. Начальный код конфигурации слоёв может выглядеть довольно громоздко.

#### Что такое паттерн Facade?
**Ответ:** Facade — это структурный паттерн проектирования, предоставляющий упрощённый интерфейс к библиотеке, фреймворку или любому другому сложному набору классов.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете изолировать свой код от сложности подсистемы.
- **Минусы:** Фасад может превратиться в god object, привязанный ко всем классам приложения.

#### Что такое паттерн Flyweight?
**Ответ:** Flyweight — это структурный паттерн проектирования, позволяющий вместить большее количество объектов в доступный объём оперативной памяти за счёт совместного использования общих частей состояния между множеством объектов вместо хранения всех данных в каждом объекте.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете сэкономить много оперативной памяти, если ваша программа работает с множеством похожих объектов.
- **Минусы:** Вы можете обменивать RAM на процессорное время, когда часть данных состояния приходится пересчитывать при каждом вызове метода flyweight. Код становится значительно сложнее. Новые члены команды всегда будут задаваться вопросом, почему состояние сущности разделено именно таким образом.

#### Что такое паттерн Proxy?
**Ответ:** Proxy — это структурный паттерн проектирования, позволяющий предоставить заместитель или заполнитель для другого объекта. Прокси контролирует доступ к оригинальному объекту, позволяя выполнить что-либо до или после того, как запрос дойдёт до оригинального объекта.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете контролировать сервисный объект незаметно для клиентов. Вы можете управлять жизненным циклом сервисного объекта, когда клиентов это не волнует. Прокси работает, даже если сервисный объект ещё не готов или недоступен. Принцип открытости/закрытости. Вы можете вводить новые прокси, не меняя сервис или клиентов.
- **Минусы:** Код может стать сложнее, так как нужно вводить множество новых классов. Ответ от сервиса может задерживаться.

#### Что такое паттерн Chain of Responsibility?
**Ответ:** Chain of Responsibility — это поведенческий паттерн проектирования, позволяющий передавать запросы по цепочке обработчиков. При получении запроса каждый обработчик решает, обработать запрос самому или передать его следующему обработчику в цепочке.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете контролировать порядок обработки запросов. Принцип единственной ответственности. Вы можете отделить классы, инициирующие операции, от классов, выполняющих операции. Принцип открытости/закрытости. Вы можете вводить новые обработчики в приложение, не ломая существующий клиентский код.
- **Минусы:** Некоторые запросы могут остаться необработанными.

#### Что такое паттерн Command?
**Ответ:** Command — это поведенческий паттерн проектирования, превращающий запрос в отдельный объект, содержащий всю информацию о запросе. Это преобразование позволяет передавать запросы как аргументы методов, откладывать или ставить в очередь выполнение запроса, а также поддерживать отменяемые операции.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип единственной ответственности. Вы можете отделить классы, инициирующие операции, от классов, выполняющих эти операции. Принцип открытости/закрытости. Вы можете вводить новые команды в приложение, не ломая существующий клиентский код. Вы можете реализовать отмену/повтор. Вы можете реализовать отложенное выполнение операций. Вы можете собирать набор простых команд в сложную.
- **Минусы:** Код может стать сложнее, так как вы вводите целый новый слой между отправителями и получателями.

#### Что такое паттерн Mediator?
**Ответ:** Mediator — это поведенческий паттерн проектирования, позволяющий уменьшить хаотичные зависимости между объектами. Паттерн ограничивает прямые связи между объектами и заставляет их взаимодействовать только через объект-посредник.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип единственной ответственности. Вы можете сосредоточить коммуникацию между различными компонентами в одном месте, упрощая понимание и поддержку. Принцип открытости/закрытости. Вы можете вводить новых посредников, не меняя сами компоненты. Вы можете снизить связанность между различными компонентами программы. Вы можете легче переиспользовать отдельные компоненты.
- **Минусы:** Со временем посредник может превратиться в God Object.

#### Что такое паттерн Iterator?
**Ответ:** Iterator — это поведенческий паттерн проектирования, позволяющий обходить элементы коллекции, не раскрывая её внутреннее представление (список, стек, дерево и т. д.).

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип единственной ответственности. Вы можете очистить клиентский код и коллекции, вынеся громоздкие алгоритмы обхода в отдельные классы. Принцип открытости/закрытости. Вы можете реализовывать новые типы коллекций и итераторов и передавать их в существующий код, ничего не ломая. Вы можете параллельно обходить одну и ту же коллекцию, так как каждый объект-итератор содержит собственное состояние обхода. По той же причине вы можете отложить обход и продолжить его, когда нужно.
- **Минусы:** Применение паттерна может быть избыточным, если ваше приложение работает только с простыми коллекциями. Использование итератора может быть менее эффективным, чем прямой доступ к элементам некоторых специализированных коллекций.

#### Что такое паттерн Memento?
**Ответ:** Memento — это поведенческий паттерн проектирования, позволяющий сохранять и восстанавливать предыдущее состояние объекта, не раскрывая деталей его реализации.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете создавать снимки состояния объекта, не нарушая его инкапсуляцию. Вы можете упростить код хозяина, позволив смотрителю вести историю состояний хозяина.
- **Минусы:** Приложение может потреблять много оперативной памяти, если клиенты слишком часто создают снимки. Смотрители должны отслеживать жизненный цикл хозяина, чтобы уничтожать устаревшие снимки. Большинство динамических языков программирования, таких как PHP, Python и JavaScript, не могут гарантировать, что состояние внутри снимка останется нетронутым.

#### Что такое паттерн Observer?
**Ответ:** Observer — это поведенческий паттерн проектирования, позволяющий определить механизм подписки для уведомления множества объектов о любых событиях, происходящих с наблюдаемым объектом.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип открытости/закрытости. Вы можете вводить новые классы подписчиков, не меняя код издателя (и наоборот, если есть интерфейс издателя). Вы можете устанавливать связи между объектами во время выполнения.
- **Минусы:** Подписчики уведомляются в произвольном порядке.

**SplObserver и SplSubject в SPL (Standard PHP Library), ссылка: https://www.php.net/manual/en/class.splobserver.php**

#### Что такое паттерн State?
**Ответ:** State — это поведенческий паттерн проектирования, позволяющий объекту изменять своё поведение при изменении внутреннего состояния. Выглядит так, будто объект сменил свой класс.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип единственной ответственности. Организация кода, связанного с конкретными состояниями, в отдельные классы. Принцип открытости/закрытости. Ввод новых состояний без изменения существующих классов состояний или контекста. Упрощение кода контекста за счёт устранения громоздких условных конструкций конечного автомата.
- **Минусы:** Применение паттерна может быть избыточным, если конечный автомат имеет мало состояний или редко меняется.

#### Что такое паттерн Strategy?
**Ответ:** Strategy — это поведенческий паттерн проектирования, позволяющий определить семейство алгоритмов, поместить каждый из них в отдельный класс и сделать их объекты взаимозаменяемыми.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете подменять алгоритмы, используемые внутри объекта, во время выполнения. Вы можете изолировать детали реализации алгоритма от кода, который его использует. Вы можете заменить наследование композицией. Принцип открытости/закрытости. Вы можете вводить новые стратегии, не меняя контекст.
- **Минусы:** Если у вас всего пара алгоритмов и они редко меняются, нет смысла усложнять программу новыми классами и интерфейсами, которые приходят вместе с паттерном. Клиенты должны знать различия между стратегиями, чтобы выбрать подходящую. Многие современные языки программирования имеют поддержку функциональных типов, что позволяет реализовать различные версии алгоритма внутри набора анонимных функций. Тогда вы можете использовать эти функции точно так же, как объекты стратегий, без раздувания кода дополнительными классами и интерфейсами.

#### Что такое паттерн Template Method?
**Ответ:** Template Method — это поведенческий паттерн проектирования, определяющий скелет алгоритма в суперклассе, но позволяющий подклассам переопределять конкретные шаги алгоритма, не меняя его структуру.

**Применение / Плюсы и минусы:**
- **Плюсы:** Вы можете позволить клиентам переопределять только определённые части большого алгоритма, делая их менее подверженными влиянию изменений в других частях алгоритма. Вы можете вынести дублирующийся код в суперкласс.
- **Минусы:** Некоторые клиенты могут быть ограничены предоставленным скелетом алгоритма. Вы можете нарушить принцип подстановки Лисков, подавляя реализацию шага по умолчанию через подкласс. Шаблонные методы, как правило, сложнее поддерживать, чем больше в них шагов.

#### Что такое паттерн Visitor?
**Ответ:** Visitor — это поведенческий паттерн проектирования, позволяющий отделить алгоритмы от объектов, над которыми они выполняются.

**Применение / Плюсы и минусы:**
- **Плюсы:** Принцип открытости/закрытости. Вы можете вводить новое поведение, работающее с объектами разных классов, не изменяя эти классы. Принцип единственной ответственности. Вы можете перенести несколько версий одного и того же поведения в один класс. Объект-посетитель может накапливать полезную информацию при работе с различными объектами. Это удобно, когда нужно обойти сложную структуру объектов, например дерево, и применить посетителя к каждому из них.
- **Минусы:** Вам нужно обновлять всех посетителей при каждом добавлении или удалении класса из иерархии элементов. Посетители могут не иметь доступа к приватным полям и методам элементов, с которыми они работают.

### Senior
#### Что такое Dependency Injection?
**Ответ:** Паттерн проектирования, при котором объект получает свои зависимости извне, а не создаёт их сам. Это способствует слабой связанности и упрощает тестирование.

#### В чём разница между Dependency Injection и Dependency Inversion?
**Ответ:** 
- **Dependency Injection (DI):** Это *паттерн* передачи зависимостей объекту извне (например, через конструктор или сеттер), вместо их создания внутри.
- **Dependency Inversion (DIP):** Это *принцип* (буква D в SOLID), согласно которому высокоуровневые модули не должны зависеть от низкоуровневых. Оба должны зависеть от абстракций (интерфейсов). DI является одним из основных способов реализации DIP.

#### Как принцип подстановки Лисков (LSP) реализуется через типизацию в PHP?
**Ответ:** 
- **Ковариантность (Covariance):** Позволяет дочернему методу возвращать *более специфичный* тип (например, `Dog` вместо `Animal`). Поддерживается с PHP 7.4.
- **Контрвариантность (Contravariance):** Позволяет дочернему методу принимать *более широкий* тип аргумента (например, `Animal` вместо `Dog`). Поддерживается с PHP 7.4.
Согласно LSP, дочерний класс не должен усиливать предусловия (контрвариантность) и не должен ослаблять постусловия (ковариантность).
[Подробный разбор SOLID](answers/solid_principles.md)

#### В чём разница между паттерном Factory и Dependency Injection?
**Ответ:** Паттерн Factory отвечает за *создание* объектов, тогда как Dependency Injection — за *предоставление* уже созданных зависимостей объекту.

#### Какие основные паттерны проектирования используются в Symfony и Doctrine?
**Ответ:**
- **Symfony:** Front Controller, Dependency Injection, Event Dispatcher.
- **Doctrine:** Data Mapper, Unit of Work, Repository.
[Паттерны проектирования в Symfony и Doctrine](answers/design_patterns.md)

## 6. Новые возможности PHP 7/8+

### Junior
#### Что такое именованные аргументы (Named Arguments) (появились в PHP 8.0)?
**Ответ:** Именованные аргументы позволяют передавать значения по имени параметра, а не по позиции, что делает код более читаемым и позволяет пропускать необязательные параметры.
```php
htmlspecialchars($string, double_encode: false);
```

#### Что такое Union Types (появились в PHP 8.0)?
**Ответ:** Union Types позволяют указывать несколько типов для параметра, свойства или возвращаемого значения вместо одного.
```php
function sum(int|float $a, int|float $b): int|float {
    return $a + $b;
}
```

#### Что такое выражения Match (появились в PHP 8.0)?
**Ответ:** Более безопасная и лаконичная альтернатива `switch`, которая использует строгое сравнение и возвращает значение.
```php
$status = match($code) {
    200, 201 => 'success',
    default => 'unknown',
};
```

#### Что такое Nullsafe Operator (появился в PHP 8.0)?
**Ответ:** Оператор `?->` позволяет безопасно обращаться к свойствам или методам потенциально null-значения без выбрасывания ошибки.
```php
$country = $session?->user?->getAddress()?->country;
```

#### Что такое атрибут `#[Override]` (появился в PHP 8.3)?
**Ответ:** Атрибут `#[Override]` гарантирует, что метод с таким же именем существует в родительском классе или интерфейсе. Это помогает выявлять ошибки при рефакторинге.
```php
class Child extends ParentClass {
    #[Override]
    public function existingMethod() {}
}
```

#### Что такое функция `json_validate()` (появилась в PHP 8.3)?
**Ответ:** Проверяет, является ли строка валидным JSON без декодирования, что потребляет меньше памяти.
```php
if (json_validate($jsonString)) {
    // строка является валидным JSON
}
```

#### Что такое динамическое получение констант класса (появилось в PHP 8.3)?
**Ответ:** Упрощённый синтаксис для получения констант класса через переменную.
```php
$name = 'STATUS_ACTIVE';
echo MyClass::{$name};
```

#### Что такое самостоятельные типы `null`, `false` и `true` (появились в PHP 8.2)?
**Ответ:** Теперь они могут использоваться как самостоятельные типы для возвращаемых значений, параметров и свойств.
```php
public function alwaysFalse(): false { return false; }
```

#### Что такое новые функции `array_*()` (появились в PHP 8.4)?
**Ответ:** В PHP 8.4 появилось несколько полезных функций для работы с массивами: `array_find()`, `array_find_key()`, `array_any()` и `array_all()`.
```php
$firstEven = array_find([1, 2, 3, 4], fn($v) => $v % 2 === 0); // 2
```

#### Что такое функции `array_first()` и `array_last()` (появились в PHP 8.5)?
**Ответ:** В PHP 8.5 добавлены нативные функции для получения первого и последнего элемента массива без перемещения внутреннего указателя массива.
```php
$arr = ['a' => 1, 'b' => 2, 'c' => 3];
$first = array_first($arr); // 1
$last = array_last($arr);   // 3
```

### Middle
#### Что такое Enums (появились в PHP 8.1)?
**Ответ:** Перечисления позволяют определить набор именованных констант, обеспечивая типобезопасность для фиксированных наборов значений.
```php
enum Status: string {
    case Pending = 'pending';
    case Active = 'active';
}
```

#### Что такое Readonly Properties (появились в PHP 8.1)?
**Ответ:** Свойства, которые не могут быть изменены после инициализации.
```php
class BlogData {
    public readonly string $title;
    public function __construct(string $title) {
        $this->title = $title;
    }
}
```

#### Что такое First-class callable syntax (появился в PHP 8.1)?
**Ответ:** Упрощённый синтаксис для получения ссылки на любую функцию или метод.
```php
$fn = strlen(...);
echo $fn('hello'); // 5
```

#### Что такое тип возврата `never` (появился в PHP 8.1)?
**Ответ:** Указывает, что функция никогда не завершится нормально (она либо выбросит исключение, либо вызовет `exit()`).
```php
function redirect(string $url): never {
    header("Location: $url");
    exit;
}
```

#### Что такое Constructor Property Promotion (появился в PHP 8.0)?
**Ответ:** Сокращённый синтаксис для объявления и инициализации свойств класса прямо в конструкторе.
```php
class User {
    public function __construct(public string $name, public int $age) {}
}
```

#### Что такое Typed Class Constants (появились в PHP 8.3)?
**Ответ:** Константы класса теперь могут иметь объявления типов, аналогично свойствам и параметрам.
```php
class User {
    const string ROLE = 'admin';
}
```

#### Что такое Asymmetric Visibility (появилась в PHP 8.4)?
**Ответ:** Свойства теперь могут иметь разную видимость для чтения и записи.
```php
class User {
    public private(set) string $name;
}
```

#### Что такое Property Hooks (появились в PHP 8.4)?
**Ответ:** Property Hooks позволяют определять пользовательскую логику получения и установки значений свойств прямо в объявлении свойства.
```php
class User {
    public string $fullName {
        get => "{$this->firstName} {$this->lastName}";
    }
}
```

readonly class Post {

public function process((A&B)|null $entity) { ... }

**Ответ:** Позволяют объявить, что параметр или возвращаемое значение должны одновременно реализовывать несколько интерфейсов или классов.
```php
public function count(Countable&Iterator $it) { ... }
```

**Ответ:** Способ создания легковесных корутин для неблокирующего ввода-вывода и асинхронных задач. Они позволяют приостанавливать и возобновлять выполнение в рамках одного потока.

#### Что такое Pipe Operator (появился в PHP 8.5)?
**Ответ:** Оператор `|>` позволяет выстраивать цепочки вызовов функций в более читаемом виде, передавая результат левой части как первый аргумент правой.
```php
$result = "  hello  " |> 'trim' |> 'strtoupper';
```

## 7. MySQL и базы данных

### Junior
#### Как подключиться к базе данных MySQL с помощью PHP?
**Ответ:** Можно использовать расширение `mysqli` или `PDO` (PHP Data Objects).
```php
// Пример с PDO
$pdo = new PDO('mysql:host=localhost;dbname=test', 'user', 'pass');
```

#### В чём разница между `mysqli` и `PDO`?
**Ответ:**
- `mysqli` работает только с MySQL, тогда как `PDO` поддерживает несколько СУБД (PostgreSQL, SQLite и др.).
- `PDO` поддерживает именованные плейсхолдеры в подготовленных запросах.

#### В чём разница между `WHERE` и `HAVING`?
**Ответ:** `WHERE` фильтрует строки *до* агрегации, тогда как `HAVING` фильтрует *после* агрегации (обычно используется с `GROUP BY`).

#### Что такое Primary Key?
**Ответ:** Первичный ключ — это столбец (или набор столбцов), уникально идентифицирующий каждую запись в таблице. Он не может быть null.

#### В чём разница между `CHAR` и `VARCHAR`?
**Ответ:** `CHAR` имеет фиксированную длину (дополняется пробелами), тогда как `VARCHAR` имеет переменную длину (использует только необходимое пространство плюс 1-2 байта на длину). `CHAR` немного быстрее для очень коротких данных одинаковой длины (например, коды стран), а `VARCHAR` экономит место для текста переменной длины.

#### Какие основные числовые типы данных есть в MySQL?
**Ответ:**
- **TINYINT:** 1 байт (от -128 до 127).
- **SMALLINT:** 2 байта (от -32 768 до 32 767).
- **INT:** 4 байта (от -2.1 млрд до 2.1 млрд).
- **BIGINT:** 8 байт.
- **DECIMAL:** Числа с фиксированной запятой для финансовых данных (точная точность).
- **FLOAT/DOUBLE:** Числа с плавающей запятой для приблизительных значений.

#### В чём разница между `DATETIME` и `TIMESTAMP`?
**Ответ:**
- **DATETIME:** 8 байт. Диапазон: от 1000-01-01 до 9999-12-31. Не зависит от часового пояса.
- **TIMESTAMP:** 4 байт. Диапазон: от 1970-01-01 до 2038-01-19. Автоматически конвертируется в UTC и обратно в зависимости от текущего часового пояса.

#### Что такое типы `ENUM` и `SET`?
**Ответ:**
- **ENUM:** Строковый объект, который может принимать только одно значение из списка разрешённых.
- **SET:** Строковый объект, который может принимать ноль или более значений из списка разрешённых (до 64).

#### В чём разница между `BLOB` и `TEXT`?
**Ответ:** Оба типа хранят большие объёмы данных, но `TEXT` предназначен для символьных строк (учитывает кодировки и правила сравнения), а `BLOB` (Binary Large Object) — для бинарных данных (изображения, файлы) и обрабатывается как строка байтов.

#### Какой порт по умолчанию у MySQL?
**Ответ:** 3306.

#### В чём разница между `CHAR_LENGTH` и `LENGTH`?
**Ответ:** `CHAR_LENGTH` считает количество символов, тогда как `LENGTH` считает количество байт.

#### Что обозначают `%` и `_` в операторе `LIKE`?
**Ответ:**
- `%` соответствует любой последовательности символов (ноль или более).
- `_` соответствует ровно одному символу.

#### Какие существуют типы соединений (Join) в MySQL?
**Ответ:**
- **(INNER) JOIN:** Возвращает записи, имеющие совпадающие значения в обеих таблицах.
- **LEFT (OUTER) JOIN:** Возвращает все записи из левой таблицы и соответствующие записи из правой таблицы.
- **RIGHT (OUTER) JOIN:** Возвращает все записи из правой таблицы и соответствующие записи из левой таблицы.
- **CROSS JOIN:** Возвращает декартово произведение двух таблиц (каждая строка первой таблицы соединяется с каждой строкой второй).
- **Self Join:** Обычное соединение, но таблица соединяется сама с собой.

#### В чём разница между неявным (Implicit) и явным (Explicit) соединением?
**Ответ:**
- **Неявное соединение:** Использует предложение `FROM` для перечисления нескольких таблиц и `WHERE` для указания условия соединения (например, `SELECT * FROM TableA, TableB WHERE TableA.id = TableB.a_id`).
- **Явное соединение:** Использует ключевое слово `JOIN` и предложение `ON` для указания условия соединения (например, `SELECT * FROM TableA JOIN TableB ON TableA.id = TableB.a_id`). Явные соединения предпочтительнее для читаемости и предотвращения случайных декартовых произведений.

#### Какие основные строковые функции есть в MySQL?
**Ответ:**
- `CONCAT(str1, str2, ...)`: Объединяет две или более строки.
- `CONCAT_WS(separator, str1, str2, ...)`: Объединяет строки с разделителем.
- `LENGTH(str)` / `CHAR_LENGTH(str)`: Возвращает длину строки в байтах/символах.
- `LEFT(str, len)` / `RIGHT(str, len)`: Возвращает указанное количество символов с начала/конца.
- `SUBSTRING(str, start, len)`: Извлекает подстроку.
- `SUBSTRING_INDEX(str, delimiter, count)`: Возвращает подстроку до указанного количества вхождений разделителя.
- `UPPER(str)` / `LOWER(str)`: Преобразует в верхний/нижний регистр.
- `TRIM(str)` / `LTRIM(str)` / `RTRIM(str)`: Удаляет начальные/конечные пробелы.
- `REPLACE(str, from_str, to_str)`: Заменяет вхождения подстроки.
- `INSERT(str, start, len, new_str)`: Заменяет часть строки другой строкой.
- `LOCATE(substr, str)`: Возвращает позицию первого вхождения подстроки.
- `REVERSE(str)`: Переворачивает строку.
- `REPEAT(str, count)`: Повторяет строку указанное количество раз.
- `LPAD(str, len, pad)` / `RPAD(str, len, pad)`: Дополняет строку до определенной длины.

#### Какие основные числовые функции есть в MySQL?
**Ответ:**
- `ABS(n)`: Возвращает абсолютное значение.
- `CEIL(n)` / `CEILING(n)`: Округляет до ближайшего целого вверх.
- `FLOOR(n)`: Округляет до ближайшего целого вниз.
- `ROUND(n, d)`: Округляет до указанного количества десятичных знаков.
- `TRUNCATE(n, d)`: Усекает до указанного количества десятичных знаков.
- `RAND()`: Возвращает случайное число с плавающей запятой от 0 до 1.
- `SQRT(n)`: Возвращает квадратный корень.
- `POWER(x, y)`: Возводит $x$ в степень $y$.
- `SIGN(n)`: Возвращает -1, 0 или 1 в зависимости от того, является ли число отрицательным, нулем или положительным.

#### Какие основные функции даты и времени есть в MySQL?
**Ответ:**
- `NOW()` / `SYSDATE()` / `CURRENT_TIMESTAMP()`: Возвращает текущую дату и время.
- `CURDATE()` / `CURRENT_DATE()`: Возвращает текущую дату.
- `CURTIME()` / `CURRENT_TIME()`: Возвращает текущее время.
- `UTC_DATE()` / `UTC_TIME()`: Возвращает текущую дату/время по UTC.
- `DATE_ADD(date, INTERVAL expr unit)` / `DATE_SUB(...)`: Добавляет/вычитает временные интервалы.
- `DATEDIFF(expr1, expr2)`: Возвращает разницу в днях между двумя датами.
- `DATE_FORMAT(date, format)`: Форматирует дату согласно шаблону.
- `EXTRACT(unit FROM date)`: Извлекает часть даты (YEAR, MONTH, DAY, HOUR, MINUTE, SECOND и т. д.).
- `DAYOFMONTH()`, `DAYOFWEEK()`, `DAYOFYEAR()`, `LAST_DAY()`: Возвращает информацию о конкретном дне.
- `MONTH()`, `YEAR()`, `QUARTER()`, `WEEK()`: Возвращает части даты.
- `HOUR()`, `MINUTE()`, `SECOND()`: Возвращает части времени.
- `TIME_TO_SEC(time)`: Преобразует время в количество секунд с начала суток.

#### Какие функции управления потоком есть в MySQL?
**Ответ:**
- `IF(expr1, expr2, expr3)`: Если `expr1` истинно, возвращает `expr2`, иначе `expr3`.
- `IFNULL(expr1, expr2)`: Если `expr1` не NULL, возвращает `expr1`, иначе `expr2`.
- `COALESCE(list)`: Возвращает первое значение из списка, не равное NULL.
- `CASE WHEN condition THEN result [ELSE result] END`: Вычисляет список условий и возвращает соответствующий результат.

#### В чём разница между `WHERE` и `HAVING`?
**Ответ:** `WHERE` фильтрует строки *до* агрегации, тогда как `HAVING` фильтрует *после* агрегации (обычно используется с `GROUP BY`).

#### В чём разница между предложением `WHERE` и условием `JOIN ... ON`?
**Ответ:**
- **`JOIN ... ON`**: Определяет, как таблицы связываются друг с другом. Обрабатывается *во время* операции соединения.
- **`WHERE`**: Фильтрует результирующие строки *после* того, как соединение было логически сформировано.
- **Производительность**: Для `INNER JOIN` они часто функционально эквивалентны, так как оптимизатор может перемещать условия между ними. Для `OUTER JOIN` (LEFT/RIGHT) расположение критично: условие в `ON` определяет, какие строки соединяются (сохраняя все строки из «внешней» таблицы), тогда как условие в `WHERE` фильтрует финальный результат (потенциально превращая LEFT JOIN в INNER JOIN, если оно отфильтровывает NULL-значения).

### Middle
#### Что такое транзакция?
**Ответ:** Последовательность операций с базой данных, которые рассматриваются как единая единица работы. Они должны соответствовать свойствам ACID.

#### Что такое свойства ACID?
**Ответ:**
- **Atomicity (Атомарность):** Всё или ничего. Гарантирует, что все операции внутри транзакции выполнены; если какая-либо не удалась, вся транзакция откатывается.
- **Consistency (Согласованность):** Гарантирует, что транзакция переводит базу данных из одного валидного состояния в другое, сохраняя все предопределённые правила.
- **Isolation (Изолированность):** Предотвращает влияние параллельных транзакций друг на друга.
- **Durability (Долговечность):** После фиксации транзакции её изменения становятся постоянными, даже в случае сбоя системы.

#### Каковы основные команды управления транзакциями?
**Ответ:**
- `START TRANSACTION` (или `BEGIN`): Начинает новую транзакцию.
- `COMMIT`: Сохраняет изменения навсегда.
- `ROLLBACK`: Отменяет изменения с начала транзакции.
- `SAVEPOINT`: Устанавливает точку внутри транзакции, к которой можно потом откатиться.
- `ROLLBACK TO SAVEPOINT`: Отменяет изменения только до указанной точки сохранения.

#### Что такое Autocommit в MySQL?
**Ответ:** Настройка, при которой каждый отдельный SQL-оператор рассматривается как транзакция и автоматически фиксируется сразу после выполнения. Можно отключить с помощью `SET autocommit = 0`.

#### Что такое Deadlock?
**Ответ:** Ситуация, когда две или более транзакции ожидают друг друга для освобождения блокировок, создавая цикл зависимостей, который не позволяет ни одной из них продолжить. MySQL обычно обнаруживает это и откатывает одну из транзакций.

#### Какие основные движки хранения существуют в MySQL?
**Ответ:**
- **InnoDB:** Поддерживает транзакции, блокировку на уровне строк и внешние ключи (по умолчанию).
- **MyISAM:** Не поддерживает транзакции, использует блокировку на уровне таблиц, быстрее для операций чтения.
- **MEMORY:** Хранит данные в RAM для очень быстрого доступа, но данные теряются при перезапуске сервера.

#### Что такое View?
**Ответ:** Виртуальная таблица, основанная на результате SQL-запроса. Она не хранит данные сама по себе.

#### Что такое Trigger?
**Ответ:** Объект базы данных, который автоматически выполняется в ответ на определённые события (INSERT, UPDATE, DELETE) на конкретной таблице.

#### Что такое InnoDB?
**Ответ:** Движок хранения для MySQL, поддерживающий транзакции, внешние ключи и блокировку на уровне строк. Является движком по умолчанию.

#### Что такое индекс и зачем он используется?
**Ответ:** Индекс — это структура данных, используемая для быстрого нахождения записей в таблице. Ускоряет запросы на чтение, но может замедлить вставки и обновления.

#### Как хранятся индексы?
**Ответ:** Большинство индексов баз данных (включая InnoDB в MySQL) хранятся как **B-деревья** (а точнее, B+Trees). Они «горизонтально широкие» (высокий fan-out) и неглубокие, что минимизирует количество операций дискового ввода-вывода для нахождения значения. Эта структура обеспечивает быструю фильтрацию (O(log n)) и эффективные диапазонные сканирования. Правильный порядок в составных индексах критически важен: как правило, столбцы с условиями равенства и высокой селективностью должны идти первыми, за ними — условия диапазона.

#### В чём разница между `UNION` и `UNION ALL`?
**Ответ:** `UNION` удаляет дублирующиеся строки из результата, тогда как `UNION ALL` включает все строки, включая дубликаты. Для использования `UNION` количество и типы столбцов в обоих операторах `SELECT` должны совпадать.

#### Как выполнять условные вычисления в запросе `SELECT`?
**Ответ:** Используйте оператор `CASE` или функцию `IF()`. Например, для начисления разных процентов в зависимости от суммы:
```sql
SELECT name, balance,
  CASE 
    WHEN balance < 3000 THEN balance * 1.1
    ELSE balance * 1.3
  END AS total
FROM accounts;
```

#### В чём разница между `IN` и `BETWEEN`?
**Ответ:**
- **IN:** Проверяет, соответствует ли значение любому значению из указанного списка или подзапроса.
- **BETWEEN:** Проверяет, попадает ли значение в указанный диапазон (включая начальное и конечное значения).

#### Что такое агрегатные функции в MySQL?
**Ответ:** Функции, которые выполняют вычисления на наборе значений и возвращают одно значение. Основные: `COUNT()`, `SUM()`, `AVG()`, `MIN()`, `MAX()`.

#### Какова цель `GROUP BY` и `HAVING`?
**Ответ:** `GROUP BY` группирует строки с одинаковыми значениями в итоговые строки. `HAVING` используется для фильтрации этих групп по условию (так как `WHERE` не может использоваться с агрегатными функциями).

#### Что такое подзапрос?
**Ответ:** Запрос, вложенный в другой запрос (SELECT, INSERT, UPDATE или DELETE). Они могут быть коррелирующими (зависящими от внешнего запроса) или некоррелирующими (независимыми).

#### Что такое оператор `EXISTS`?
**Ответ:** Логический оператор, используемый в предложении `WHERE` для проверки, возвращает ли подзапрос какие-либо строки. Он возвращает true, если подзапрос возвращает одну или более записей. Часто он эффективнее, чем `IN`, для проверки существования.

#### В чём разница между `ANY` и `ALL` при использовании с подзапросами?
**Ответ:**
- **ANY (или SOME):** Возвращает true, если условие сравнения истинно хотя бы для *одного* значения из результата подзапроса.
- **ALL:** Возвращает true только в том случае, если условие сравнения истинно для *всех* значений из результата подзапроса.

#### Какие существуют общие ограничения (атрибуты) столбцов?
**Ответ:**
- **NOT NULL:** Гарантирует, что столбец не может содержать значение NULL.
- **UNIQUE:** Гарантирует, что все значения в столбце различны.
- **PRIMARY KEY:** Комбинация NOT NULL и UNIQUE.
- **FOREIGN KEY:** Предотвращает действия, которые могут разрушить связи между таблицами.
- **DEFAULT:** Устанавливает значение по умолчанию для столбца, если оно не указано.
- **CHECK:** Гарантирует, что значения в столбце соответствуют определённому условию.
- **AUTO_INCREMENT:** Автоматически генерирует уникальное число для каждой новой строки.

#### Каковы действия `ON DELETE` и `ON UPDATE` для внешних ключей?
**Ответ:**
- **CASCADE:** Автоматически удаляет или обновляет строки в дочерней таблице при удалении/обновлении строки в родительской таблице.
- **SET NULL:** Устанавливает значение NULL в столбце внешнего ключа в дочерней таблице.
- **RESTRICT / NO ACTION:** Отклоняет операцию удаления или обновления в родительской таблице, если существуют связанные строки в дочерней таблице.

#### Для чего используется оператор `CONSTRAINT`?
**Ответ:** Он используется для определения и именования ограничений (PRIMARY KEY, UNIQUE, FOREIGN KEY, CHECK) таблицы. Именование ограничений позволяет легко удалять или изменять их в дальнейшем, используя эти имена.

#### В чём разница между `LIKE` и `REGEXP`?
**Ответ:**
- **LIKE:** использует простые подстановочные знаки (`%` для любой строки, `_` для любого одиночного символа).
- **REGEXP:** использует мощные регулярные выражения (`^`, `$`, `.`, `[ ]`, `|` и т. д.) для сложного сопоставления с шаблоном.

#### Как можно изменить структуру существующей таблицы?
**Ответ:** С помощью команды `ALTER TABLE` для добавления, удаления или изменения столбцов и ограничений.

#### В чём разница между MariaDB и MySQL?
**Ответ:** MariaDB — это форк MySQL с открытым исходным кодом, созданный первоначальными разработчиками после приобретения Sun Microsystems (владевшей MySQL) компанией Oracle. Сохраняя высокую совместимость, MariaDB предлагает больше движков хранения, улучшения производительности и остаётся полностью открытой (GPL), тогда как MySQL имеет как community, так и enterprise версию.
[Сравнение MariaDB и MySQL](answers/mysql_advanced.md#9-mariadb-vs-mysql)

### Senior
#### Что такое Sharding и Partitioning?
**Ответ:**
- **Partitioning:** Разделение большой таблицы на более мелкие, управляемые части в рамках одного сервера базы данных.
- **Sharding:** Распределение данных по нескольким независимым серверам баз данных (горизонтальное масштабирование).

#### В чём разница между Clustered Index и Non-Clustered Index?
**Ответ:**
- **Clustered Index:** Определяет физический порядок данных в таблице. Может быть только один на таблицу (обычно это первичный ключ).
- **Non-Clustered Index:** Отдельная структура, содержащая индексированные значения и указатели на строки данных.

#### Какие уровни изоляции вы знаете?
**Ответ:** Стандартные уровни изоляции: Read Uncommitted, Read Committed, Repeatable Read и Serializable. Они определяют баланс между согласованностью и производительностью при параллельных транзакциях.
[Подробное руководство по уровням изоляции](answers/mysql_advanced.md#5-transaction-isolation-levels)

#### В чём разница между Read Committed и Dirty Read?
**Ответ:** Dirty Read возникает при уровне Read Uncommitted, когда транзакция видит незафиксированные данные другой транзакции. Read Committed предотвращает это, гарантируя, что транзакция видит только данные, которые уже были зафиксированы.
[Подробное руководство по уровням изоляции](answers/mysql_advanced.md#5-transaction-isolation-levels)

#### Приведите примеры уровней изоляции транзакций и проблемы, которые они решают.
**Ответ:**
- **Read Uncommitted:** Чтение баланса банковского счёта, который обновляется, но ещё не зафиксирован (Dirty Read).
- **Read Committed:** Отчёт читает одни и те же данные дважды и получает разные результаты, потому что другая транзакция зафиксировала обновление между чтениями (Non-Repeatable Read).
- **Repeatable Read:** Обеспечение согласованного снимка данных на протяжении длительной транзакции, даже если другие транзакции фиксируют изменения.
- **Serializable:** Финансовые системы, где требуется абсолютная корректность, предотвращение всех проблем параллельности, включая Phantom Reads, ценой производительности.
[Подробное руководство по уровням изоляции](answers/mysql_advanced.md#51-practical-examples-of-isolation-levels)

#### В чём разница между MySQL и PostgreSQL?
**Ответ:** MySQL известен своей скоростью и простотой использования в веб-разработке, тогда как PostgreSQL — это объектно-реляционная база данных, известная расширенными возможностями, строгим соответствием SQL и лучшей обработкой сложных запросов и высоконагруженных параллельных рабочих процессов.
[Сравнение MySQL и PostgreSQL](answers/mysql_advanced.md#10-mysql-vs-postgresql)

#### У нас есть фильтр по 3 столбцам из одной таблицы: `sku` (уникальный), `creation_date` и `integer_field` (значения 1, 2 или 3). Как бы вы индексировали это для максимальной производительности?
**Ответ:** Создайте **составной индекс**, включающий все три поля. Для максимальной производительности порядок должен быть: **sku** (наибольшая селективность), затем **integer_field** (точные совпадения) и наконец **creation_date** (диапазонные фильтры). Такой порядок обеспечивает, что наиболее селективный фильтр сужает результаты немедленно, а диапазонный фильтр идёт последним, позволяя предшествующим частям индекса быть полностью использованными.
[Стратегия составных индексов](answers/mysql_advanced.md#11-composite-index-strategy)

#### У вас есть этот SQL-запрос. Что в нём не так и как бы вы его улучшили?
`SELECT * FROM table ORDER BY id LIMIT :limit OFFSET :offset`
- `id` — это Primary Key, AutoIncrement и проиндексирован.

**Ответ:**
Использование `OFFSET` для пагинации неэффективно для больших наборов данных, потому что MySQL должен просканировать и отбросить все строки до смещения. Например, `OFFSET 1000000` требует чтения 1 миллиона строк перед возвратом запрошенных, что приводит к высокой нагрузке на I/O и CPU.

**Оптимизация (пагинация на основе курсора):**
- **Начальный запрос:** `SELECT * FROM table ORDER BY id LIMIT :limit`
- **Последующие запросы:** Получите **последний ID** с текущей страницы (например, `:last_id`) и используйте его для пропуска предыдущих строк:
  `SELECT * FROM table WHERE id > :last_id ORDER BY id LIMIT :limit`
Это гарантирует, что запрос остаётся быстрым вне зависимости от номера страницы, так как использует индекс первичного ключа для перехода прямо к начальной точке.

#### У вас есть эти SQL-запросы, какие индексы вы бы использовали?
1. `SELECT * FROM transactions WHERE account_id = :account_id AND transaction_type = :transaction_type AND created_at BETWEEN :start AND :end`
2. `SELECT * FROM transactions WHERE account_id = :account_id AND transaction_type = :transaction_type ORDER BY created_at`
- `account_id`: bigint, ~100 000 вариантов
- `transaction_type`: int, 15 вариантов
- `created_at`: DATETIME, DEFAULT NOW()

**Ответ:**
Используйте составной индекс: `idx (account_id, transaction_type)`.

**Обоснование:**
- Лучше сначала фильтровать по `account_id`, чтобы значительно сузить область (высокая селективность).
- Затем фильтровать по `transaction_type`.
- Во втором запросе `created_at` используется для сортировки. Хотя добавление его в индекс может избежать `filesort`, включение не всегда целесообразно, если он не используется для значительного сужения данных в большинстве запросов, или если отфильтрованный набор результатов достаточно мал для эффективной сортировки после выборки.

#### У вас есть этот код, есть ли в нём проблемы?
```php
public function addBalance(float $sum, User $user) {
    $balance = $this->repo->getUserBalance($user->getId());
    $balance += $sum;
    $this->repo->updateUserBalance($balance, $user->getId());
}
// В getUserBalance: SELECT balance FROM Users WHERE id = :id
// В updateUserBalance: UPDATE Users SET balance = :balance WHERE id = :id
```

**Ответ:**
Этот код уязвим к **состояниям гонки** (Lost Updates). Если два параллельных процесса прочитают один и тот же баланс до того, как один из них обновит его, одно обновление перезапишет другое.

**Подробное объяснение:**
- На самом слабом уровне изоляции (**Read Uncommitted**) может возникнуть **Dirty Read** (чтение незафиксированных данных).
- Даже при уровнях изоляции по умолчанию параллельные циклы «чтение-модификация-запись» приводят к потере данных.
- **Решение:** Используйте **`SELECT ... FOR UPDATE`** в MySQL. Это блокирует конкретную строку, заставляя другие транзакции ждать, пока блокировка не будет снята (после фиксации `UPDATE`).

**Корректная реализация:**
```php
$this->db->transaction(function() use ($sum, $user) {
    // SELECT balance FROM Users WHERE id = :id FOR UPDATE
    $balance = $this->repo->getUserBalanceForUpdate($user->getId());
    $balance += $sum;
    $this->repo->updateUserBalance($balance, $user->getId());
});
```

#### Можете ли вы привести конкретные примеры SQL, показывающие поведение различных уровней изоляции и блокировок в условиях параллельного доступа?
**Ответ:**
Понимание точного поведения уровней изоляции и блокировок критически важно для отладки состояний гонки. Подробные пошаговые сценарии (включая временные линии T1/T2) для Dirty Reads, Non-Repeatable Reads и Phantom Reads, а также конкретные примеры Exclusive (`FOR UPDATE`) и Shared (`LOCK IN SHARE MODE`) блокировок и Deadlock-ов представлены в руководстве.
[Подробные сценарии изоляции и блокировок в MySQL](answers/mysql_advanced.md#52-detailed-isolation-level-scenarios)

#### Как обеспечить идемпотентность при вставке записей (например, в консьюмере брокера сообщений)?
**Ответ:** Используйте **Unique Constraints** на бизнес-ключах (например, `(order_id, user_id)`). Когда сообщение будет повторно доставлено (гарантия "at-least-once"), база данных заблокирует вторую вставку. Ваше приложение должно перехватывать ошибку `Duplicate Key` и обрабатывать её как успех или известное состояние, чтобы гарантировать корректный результат для клиента.

#### Как управлять лимитами (например, максимальным количеством участников) в условиях высокой нагрузки?
**Ответ:**
1.  **Пессимистическая блокировка**: `SELECT * FROM meetups WHERE id = 1 FOR UPDATE;` для сериализации всех регистраций на конкретный митап.
2.  **Денормализация + CHECK Constraint**: Добавьте колонку `booked_seats` и ограничение `CHECK (booked_seats <= total_seats)`. Обновляйте счетчик в той же транзакции, что и регистрация. База данных сама обеспечит соблюдение лимита.
[Подробное руководство по конкурентности и лимитам в SQL](answers/sql_concurrency.md)

#### Что такое `xmin` и `xmax` в контексте строк базы данных (например, в PostgreSQL)?
**Ответ:** Это скрытые системные колонки, используемые для реализации **MVCC (Multi-Version Concurrency Control)**.
- **PostgreSQL**:
  - `xmin`: хранит ID транзакции, которая вставила строку.
  - `xmax`: хранит ID транзакции, которая удалила или обновила строку.
- **MySQL (InnoDB)**:
  - `DB_TRX_ID`: аналогично `xmin`, хранит ID транзакции, которая создала версию строки.
  - `DB_ROLL_PTR`: указатель на предыдущую версию в **Undo Log** (используется для восстановления старых версий).
  - `DB_ROW_ID`: скрытый ID строки (если нет первичного ключа).
Понимание этих колонок помогает объяснить, как разные транзакции «видят» разные версии данных одновременно без блокировки друг друга.

#### Когда следует использовать блокировки Redis вместо встроенных возможностей SQL?
**Ответ:** Используйте Redis для **Rate Limiting** (ограничение частоты запросов), **Circuit Breakers** (предохранители) или **синхронизации задач между узлами** (например, запуск Cron один раз). Избегайте использования Redis для обеспечения консистентности данных *внутри* БД. SQL-функции (уникальные индексы, ограничения, блокировки строк) более надежны, лучше справляются с отказами (нет проблем с TTL) и не создают лишних точек отказа или накладных расходов на уровне данных.

## 8. PostgreSQL

### Junior

#### Что такое PostgreSQL и чем она отличается от MySQL?
**Ответ:** PostgreSQL — это мощная объектно-реляционная система управления базами данных (ORDBMS) с открытым исходным кодом. В отличие от MySQL, которая является чисто реляционной, PostgreSQL поддерживает расширенные типы данных (такие как массивы и геометрические типы) и известна своим строгим соответствием стандартам SQL и расширяемостью.
[Сравнение MySQL и PostgreSQL](answers/mysql_vs_postgresql.md)

#### Какие основные числовые типы данных есть в PostgreSQL?
**Ответ:** 
- **целые числа:** `smallint` (2 байта), `integer` (4 байта), `bigint` (8 байт).
- **числа с плавающей точкой:** `real` (4 байта), `double precision` (8 байт).
- **числа с фиксированной точкой:** `numeric` или `decimal` (точная точность для финансовых данных).
- **автоинкремент:** `serial`, `smallserial`, `bigserial`.
[Подробные типы данных PostgreSQL](answers/postgresql_data_types.md)

#### В чём разница между `VARCHAR(n)`, `CHAR(n)` и `TEXT` в PostgreSQL?
**Ответ:** 
- `CHAR(n)`: фиксированная длина, дополняется пробелами.
- `VARCHAR(n)`: переменная длина с максимальным ограничением.
- `TEXT`: переменная длина без конкретного ограничения.
В PostgreSQL нет разницы в производительности между этими тремя типами, поэтому для гибкости часто предпочитают `TEXT` или `VARCHAR` (без указания `n`).

#### Как определить первичный ключ (Primary Key) в PostgreSQL?
**Ответ:** Вы можете использовать `PRIMARY KEY` на уровне столбца или таблицы. Он однозначно идентифицирует каждую строку и не может быть NULL.
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username TEXT NOT NULL
);
```

#### Что такое внешний ключ (Foreign Key) и как он используется?
**Ответ:** Внешний ключ связывает столбец в одной таблице с первичным ключом другой таблицы. Он обеспечивает ссылочную целостность.
```sql
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE
);
```

#### В чём разница между `INNER JOIN` и `LEFT JOIN`?
**Ответ:** `INNER JOIN` возвращает только совпадающие строки из обеих таблиц. `LEFT JOIN` возвращает все строки из левой таблицы и совпадающие строки из правой таблицы (с NULL, если совпадений нет).
[Связи между таблицами в PostgreSQL](answers/postgresql_relations.md)

- `CHECK`: гарантирует, что значения в столбце удовлетворяют определённому логическому условию (например, `price > 0`).

### Senior

#### Что такое MVCC (Multi-Version Concurrency Control) in PostgreSQL?
**Ответ:** MVCC позволяет нескольким транзакциям видеть согласованный снимок базы данных, не блокируя друг друга. Он использует скрытые столбцы, такие как `xmin` and `xmax`, для отслеживания версий строк.

#### Что такое коррелирующие подзапросы (Correlated Subqueries)?
**Ответ:** Подзапрос, который ссылается на столбцы внешнего запроса. Он выполняется один раз для каждой строки, обрабатываемой внешним запросом.
```sql
SELECT name, (SELECT AVG(price) FROM products p2 WHERE p2.category = p1.category)
FROM products p1;
```

#### Как работают `GROUPING SETS`, `ROLLUP` и `CUBE`?
**Ответ:** Это расширения `GROUP BY` для создания нескольких наборов группировок в одном запросе (полезно для отчётов и аналитики).
- `ROLLUP`: генерирует иерархические промежуточные итоги.
- `CUBE`: генерирует промежуточные итоги для всех возможных комбинаций столбцов.
- `GROUPING SETS`: позволяет указать точные наборы столбцов для группировки.

#### Как можно изменить структуру таблицы с помощью `ALTER TABLE`?
**Ответ:** Вы можете добавлять/удалять столбцы, изменять типы и добавлять/удалять ограничения.
```sql
ALTER TABLE users ADD COLUMN email TEXT UNIQUE;
ALTER TABLE users ALTER COLUMN age TYPE SMALLINT;
```

## 9. Laravel и Symfony

### Junior
#### Что такое архитектура MVC и как Laravel её реализует?
**Ответ:** MVC расшифровывается как Model-View-Controller.
- **Model:** Представляет структуру данных (Eloquent).
- **View:** Отображает данные (шаблоны Blade).
- **Controller:** Обрабатывает логику и пользовательский ввод.

#### Что такое Eloquent ORM?
**Ответ:** Встроенный в Laravel объектно-реляционный преобразователь (ORM), позволяющий взаимодействовать с базой данных через PHP-объекты вместо чистого SQL.

#### Что такое Blade?
**Ответ:** Blade — это мощный и простой шаблонизатор Laravel, позволяющий использовать чистый PHP в шаблонах с лаконичным синтаксисом.

#### Что такое маршрутизация (Routing) в Laravel?
**Ответ:** Сопоставление HTTP-запросов (URL-адресов) с конкретными действиями контроллеров или замыканиями.

#### Что такое именованные маршруты (Named Routes)?
**Ответ:** Присвоение имени маршруту для генерации URL-адресов или перенаправлений по имени, а не по сырому URL.

#### Что такое группы маршрутов (Route Groups)?
**Ответ:** Способ группировки нескольких маршрутов с общими атрибутами (такими как middleware или префиксы).

#### Что такое Seeders и Factories?
**Ответ:**
- **Seeders:** Заполняют базу данных тестовыми/фиктивными данными.
- **Factories:** Определяют шаблон для генерации фиктивных данных модели.

#### Что такое Soft Delete?
**Ответ:** Пометка записи как удалённой без фактического удаления из базы данных, обычно с использованием столбца `deleted_at`.

### Middle
#### Что такое Laravel Service Container?
**Ответ:** Мощный инструмент для управления зависимостями классов и выполнения Dependency Injection.
[Подробный разбор Service Container](answers/laravel_service_container.md)

#### Что такое Collections?
**Ответ:** Обёртка над PHP-массивами, предоставляющая мощный, удобный интерфейс для манипуляции данными (например, `map`, `filter`, `reduce`).

#### Что такое Middleware?
**Ответ:** Фильтр, инспектирующий и фильтрующий HTTP-запросы, поступающие в ваше приложение (например, аутентификация, логирование).

#### Что такое Service Providers?
**Ответ:** Центральное место начальной загрузки всего приложения Laravel. Они привязывают сервисы к Service Container.

#### Что такое Migrations?
**Ответ:** Система контроля версий для базы данных, позволяющая определять структуру таблиц в PHP-коде.

#### В чём разница между аутентификацией и авторизацией?
**Ответ:**
- **Аутентификация:** Проверка *кто* является пользователем (например, вход в систему).
- **Авторизация:** Проверка *что* пользователю разрешено делать (например, права доступа).

### Senior
#### Что такое Facades в Laravel?
**Ответ:** Они предоставляют «статический» интерфейс к классам, доступным в Service Container. Служат «прокси»-классами для доступа к базовым реализациям.

#### Что такое Events и Listeners?
**Ответ:** Способ разделения компонентов приложения. Событие (Event) отправляется, и один или несколько слушателей (Listeners) реагируют на него.

#### Что такое Contracts в Laravel?
**Ответ:** Набор интерфейсов, определяющих основные сервисы, предоставляемые фреймворком. Они позволяют подменять базовые реализации.

#### Каковы роли Service Container и Service Provider?
**Ответ:**
- **Service Container:** Реестр всех зависимостей.
- **Service Provider:** Классы, которые регистрируют и загружают эти зависимости в контейнер.

## 13. Тестирование и качество

### Junior
#### Почему важно писать тесты?
**Ответ:** Тесты гарантируют, что код работает как ожидается, предотвращают регрессии (ошибки, появляющиеся после изменений), документируют поведение кода и делают рефакторинг безопаснее. Они также помогают проектировать лучшую архитектуру, поощряя модульность.

#### Что такое Unit Tests?
**Ответ:** Тип тестирования ПО, при котором отдельные модули или компоненты тестируются изолированно. В PHP стандартным инструментом является PHPUnit.

#### Какие фреймворки для Unit Tests вы использовали?
**Ответ:** Основной фреймворк для PHP — **PHPUnit**. Также использовал **Pest**, который построен поверх PHPUnit и предлагает более выразительный функциональный синтаксис.

#### Что такое инструменты статического анализа?
**Ответ:** Инструменты статического анализа анализируют код без его выполнения. Они помогают находить потенциальные ошибки, уязвимости безопасности, нарушения стиля кода и другие проблемы на ранних этапах разработки.

#### Как измеряется покрытие тестами?
**Ответ:** Покрытие тестами (Code Coverage) — это метрика, измеряющая процент строк кода, выполняемых набором тестов. Оно измеряется отслеживанием, какие части кода затрагиваются при выполнении тестов, обычно с помощью инструментов **Xdebug** или **PCOV**.

### Middle
#### Какие инструменты статического анализа вы использовали?
**Ответ:** Использовал **PHPStan** для поиска ошибок и улучшения качества кода, **Psalm** для расширенного статического анализа (особенно для больших кодовых баз) и **PHP_CodeSniffer (PHPCS)** для поддержания единообразия стиля кода.

#### В чём разница между unit, integration и feature тестами?
**Ответ:**
- **Unit-тесты:** Тестируют изолированные компоненты (например, один метод) в отрыве от зависимостей (часто с использованием моков).
- **Integration-тесты:** Тестируют взаимодействие между несколькими компонентами (например, как сервис взаимодействует с репозиторием или базой данных).
- **Feature-тесты:** Тестируют полный пользовательский сценарий или API-эндпоинт от начала до конца, часто затрагивая несколько слоёв приложения.

#### Какой метод измерения покрытия unit-тестами вы использовали?
**Ответ:** Широко используемые методы включают **Line Coverage** (процент выполненных строк), **Branch Coverage** (проверка всех путей через условия) и **Path Coverage**. PHPUnit обычно выводит покрытие строк с использованием драйверов Xdebug или PCOV.

#### Что такое интеграционное тестирование?
**Ответ:** Тестирование того, как различные модули или компоненты приложения работают вместе как группа.

#### Что такое Mocking?
**Ответ:** Создание поддельной версии объекта, имитирующей поведение реального объекта, используется для изоляции тестируемого модуля кода.

#### TDD vs BDD
**Ответ:**
- **TDD (Test-Driven Development):** Написание тестов *до* написания фактического кода. Фокусируется на реализации.
- **BDD (Behavior-Driven Development):** Написание тестов на основе ожидаемого поведения и пользовательских сценариев (например, Behat). Фокусируется на коммуникации.

### Senior
#### Что такое Mutation Testing?
**Ответ:** Тип тестирования, при котором в код вносятся небольшие изменения (мутации) для проверки того, что существующие тесты обнаруживают их. Это измеряет эффективность тестов.

## 14. Безопасность

### Junior
#### В чём разница между хешированием и шифрованием?
**Ответ:**
- **Хеширование:** Односторонняя функция, создающая строку фиксированной длины из входных данных. Не может быть обращена (например, `password_hash`).
- **Шифрование:** Двусторонняя функция, при которой данные могут быть преобразованы в шифротекст, а затем расшифрованы обратно в открытый текст с помощью ключа.

#### Зачем используется соль (Salt) при хешировании паролей?
**Ответ:** Для предотвращения использования злоумышленниками предварительно вычисленных таблиц (таких как Rainbow Tables) для взлома паролей. Уникальная соль добавляется к каждому паролю перед хешированием.

### Middle
#### Что такое SQL Injection и как его предотвратить?
**Ответ:** Уязвимость, позволяющая злоумышленнику выполнять вредоносные SQL-запросы. Предотвращается использованием подготовленных выражений и параметризованных запросов.

#### Что такое XSS (Cross-Site Scripting) и как его предотвратить?
**Ответ:** Атака, при которой вредоносные скрипты внедряются в веб-страницы. Предотвращается экранированием вывода (с помощью `htmlspecialchars`) и санитизацией пользовательского ввода.

#### Что такое CSRF (Cross-Site Request Forgery) и как его предотвратить?
**Ответ:** Атака, вынуждающая аутентифицированного пользователя выполнить нежелательные действия в веб-приложении. Предотвращается использованием CSRF-токенов.

#### Что такое JWT (JSON Web Token)?
**Ответ:** Компактный, URL-безопасный способ представления утверждений (claims) для передачи между двумя сторонами. Часто используется для stateless-аутентификации.

### Senior
#### Что такое OWASP Top 10?
**Ответ:** Стандартный документ для повышения осведомлённости разработчиков и специалистов по безопасности веб-приложений, представляющий широкий консенсус относительно наиболее критичных рисков безопасности веб-приложений.

## 15. Веб и API

### Junior
#### Что такое REST API?
**Ответ:** REST (Representational State Transfer) — это архитектурный стиль для проектирования сетевых приложений. Он опирается на stateless, клиент-серверный, кешируемый протокол коммуникации — и практически во всех случаях используется протокол HTTP.
[Подробнее: Что такое REST API?](https://www.ibm.com/think/topics/rest-apis)
#### Какие основные HTTP-методы используются в REST?
**Ответ:**
- `GET`: Получить ресурс.
- `POST`: Создать новый ресурс.
- `PUT`: Обновить/заменить существующий ресурс.
- `PATCH`: Частично обновить существующий ресурс.
- `DELETE`: Удалить ресурс.
- `OPTIONS`: Описать доступные варианты взаимодействия с целевым ресурсом.
- `HEAD`: То же, что GET, но возвращает только заголовки (без тела).
#### В чём разница между GET и POST?
**Ответ:**
- `GET`: Данные передаются в URL. Используется для получения данных.
- `POST`: Данные передаются в теле запроса. Используется для отправки данных.

#### Что такое HTTP-коды состояния? Приведите примеры.
**Ответ:**
- `200 OK`: Успех.
- `201 Created`: Ресурс успешно создан.
- `400 Bad Request`: Ошибка клиента.
- `401 Unauthorized`: Пользователь не аутентифицирован.
- `403 Forbidden`: Пользователь аутентифицирован, но доступ запрещён.
- `404 Not Found`: Ресурс не найден.
- `500 Internal Server Error`: Ошибка сервера.

### Middle
#### Что означает "stateless" в контексте REST?
**Ответ:** Каждый запрос от клиента к серверу должен содержать всю необходимую информацию для понимания и обработки запроса. Сервер не хранит контекст клиента между запросами.

#### SOAP vs REST
**Ответ:**
- **SOAP:** Основан на протоколе, использует XML, имеет жёсткую структуру, хорош для безопасности и соответствия ACID.
- **REST:** Архитектурный стиль, использует JSON/XML, гибкий, легковесный и использует стандартный HTTP.

#### REST vs JSON-RPC
**Ответ:**
- **REST:** Ориентирован на ресурсы, использует HTTP-методы (GET, POST и т. д.) и коды состояния.
- **JSON-RPC:** Ориентирован на методы, обычно использует POST на один эндпоинт с JSON-полезной нагрузкой, указывающей метод и параметры.

#### Можно ли реализовать собственный HTTP-метод, например POSTAWESOME, вместо POST? Будет ли это соответствовать REST?
**Ответ:**
Технически — да, вы можете определить и использовать пользовательские HTTP-методы. Протокол HTTP допускает расширения, и большинство веб-серверов и клиентов могут быть настроены для их обработки.
Однако это **не** будет полностью соответствовать ограничению **Uniform Interface** REST. REST подразумевает стандартизированный набор методов (стандартные HTTP-глаголы) для обеспечения интероперабельности, предсказуемости и использования существующей веб-инфраструктуры (кеши, прокси), которая понимает только стандартные методы. Использование `POSTAWESOME` нарушит эти преимущества и затруднит потребление вашего API.

## 16. Высокие нагрузки и масштабирование

### Middle
#### Что такое Load Balancing?
**Ответ:** Распределение сетевого или прикладного трафика между несколькими серверами для повышения отзывчивости и доступности.

### Senior
#### Как оптимизировать медленный GET-эндпоинт, возвращающий много записей?
**Ответ:**
- Использовать пагинацию.
- Использовать кеширование (Redis/Memcached).
- Оптимизировать запросы к базе данных (индексы, выбор только нужных столбцов).
- Использовать Eager Loading для предотвращения проблемы N+1.

#### Что такое CAP-теорема?
**Ответ:** В распределённой системе можно обеспечить только два из трёх гарантий: Consistency (согласованность), Availability (доступность) и Partition Tolerance (устойчивость к разделению).

#### Что такое репликация базы данных?
**Ответ:** Копирование данных с одного сервера базы данных (Master) на другой (Slave) для повышения надёжности, отказоустойчивости и производительности.

## 17. Чистый код и лучшие практики

### Junior
#### Что такое DRY и KISS?
**Ответ:**
- **DRY:** Don't Repeat Yourself (Не повторяйся).
- **KISS:** Keep It Simple, Stupid (Не усложняй).

#### Что такое YAGNI?
**Ответ:** You Ain't Gonna Need It (Вам это не понадобится). Не реализуйте функциональность, пока она действительно не нужна.

### Middle
#### Что такое Composition over Inheritance?
**Ответ:** Принцип, предлагающий достигать полиморфного поведения и повторного использования кода путём композиции объектов с другими объектами, реализующими нужную функциональность, вместо наследования от базового или родительского класса.

### Senior
#### Представьте, что у вас есть старый класс в монолите, этот класс имеет много методов и нуждается в изменении. Как бы вы его отрефакторили?
**Ответ:**
1. **Понять и покрыть тестами:** Напишите тесты для существующей функциональности, чтобы гарантировать отсутствие регрессий при рефакторинге.
2. **Определить ответственности:** Выявите ключевые обязанности класса и выделите их в отдельные классы с единой ответственностью (SRP).
3. **Разделить зависимости:** Используйте Dependency Injection для отвязки класса от его зависимостей, делая его более тестируемым.
4. **Инкрементальный рефакторинг:** Рефакторьте маленькими шагами, запуская тесты после каждого изменения.
5. **Применить паттерны проектирования:** Используйте паттерны Strategy, Factory или Service-объекты для обработки сложной логики или множественных условных путей.
[Подробное руководство по паттернам проектирования](answers/design_patterns.md)

#### Что такое закон Деметры?

## 19. Каверзные вопросы

### Junior
#### В чём разница между `==` и `===`?
**Ответ:**
- `==`: Сравнение (проверяет значение, выполняет приведение типов).
- `===`: Тождество (проверяет и значение, и тип).

#### Что происходит при сравнении "10" и 10 с помощью `==`?
**Ответ:** Возвращает `true` из-за приведения типов (type juggling).

#### Что выведет следующий код?
```php
function foo(&$var) {
    $var++;
}
$a = 5;
foo($a);
echo $a;
```
**Ответ:** `6`. Оператор `&` передаёт переменную по ссылке, поэтому изменение внутри функции влияет на оригинальную переменную.

### Middle
#### В чём разница между `array_merge()` и оператором `+` для массивов?
**Ответ:**
- `array_merge()`: Для строковых ключей последнее значение перезаписывает предыдущее. Для числовых ключей значения добавляются в конец и переиндексируются.
- `+`: Для строковых ключей первое значение сохраняется, последующие игнорируются. Для числовых ключей также сохраняется первое.

#### Как ведут себя статические свойства в наследуемых классах?
**Ответ:** Статические свойства разделяются между всеми классами в иерархии наследования, если они явно не переопределены в дочернем классе.

### Senior
#### Почему использование PHP в качестве демона является рискованным?
**Ответ:** PHP изначально проектировался для короткоживущих запросов. В долгоживущем процессе утечки памяти в вашем коде или библиотеках со временем приведут к аварийному завершению процесса.

## 21. Long-Running (RoadRunner)

### Middle
#### Что такое RoadRunner и как он работает?
**Ответ:** RoadRunner — это высокопроизводительный PHP application server, балансировщик нагрузки и менеджер процессов, написанный на Go. Он работает как постоянный процесс, управляющий пулом PHP-воркеров. Когда приходит запрос, RoadRunner передаёт его свободному PHP-воркеру через высокопроизводительный бинарный протокол (Goridge). PHP-воркер остаётся живым после завершения запроса, устраняя накладные расходы на повторную инициализацию движка PHP и фреймворка приложения (bootstrapping) при каждом запросе.
[Подробное руководство по RoadRunner](answers/roadrunner.md)
![Как это работает](https://docs.roadrunner.dev/images/rr-scheme.png)

#### Что такое PHP Worker в контексте RoadRunner?
**Ответ:** PHP Worker — это скрипт, работающий в непрерывном цикле. Он использует класс `Spiral\RoadRunner\Worker` для ожидания запросов от сервера RoadRunner, их обработки (часто через PSR-7) и отправки ответов обратно. Воркеры могут взаимодействовать с сервером через стандартные каналы ввода-вывода (по умолчанию), TCP или Unix-сокеты. Начиная с RoadRunner 2.0, любые предупреждения или вывод в `STDOUT` автоматически перенаправляются в `STDERR` для предотвращения повреждения протокола.

#### Как RoadRunner управляет пулом воркеров?
**Ответ:** RoadRunner поддерживает пул PHP-воркеров и может динамически масштабировать их. Он использует стратегию супервизии для перезапуска воркеров на основе различных лимитов, определённых в `.rr.yaml`:
- `max_jobs`: Количество запросов, которое воркер может обработать до замены.
- `max_memory`: Лимит памяти на воркер.
- `ttl`: Максимальное время жизни воркера.
- `idle_ttl`: Максимальное время простоя воркера.
Для локальной разработки можно использовать `pool.debug = true` для выделения воркера только при поступлении запроса, что упрощает отладку.

#### Что нужно учитывать при написании кода для долгоживущего PHP-окружения?
**Ответ:**
- **Утечки памяти:** Любая утёкшая память будет накапливаться со временем; используйте `supervisor.max_worker_memory` как защиту.
- **Сохранение состояния:** Статические переменные и глобальное состояние сохраняются между запросами, что может привести к утечкам данных или неожиданному поведению, если не сбрасывать их.
- **Управление ресурсами:** Подключения к базе данных и файловые дескрипторы должны перепроверяться или правильно закрываться, так как они могут истекать по таймауту или оставаться открытыми на протяжении многих запросов.
- **Обработка ошибок:** Необработанные исключения могут аварийно завершить воркер; их следует перехватывать и передавать обратно в RoadRunner через `$worker->error()`.

#### Что такое плагин Key-Value (KV) в RoadRunner?
**Ответ:** Плагин KV предоставляет единый интерфейс для взаимодействия с различными хранилищами для простого key-value хранения. Он переносит кеширование или управление состоянием из PHP на Go-сервер. Поддерживаемые драйверы:
- `redis` (распределённый)
- `memcached` (распределённый)
- `boltdb` (файловый)
- `memory` (быстрый, локальный для экземпляра RR)

#### Как работают очереди и задачи (Jobs) в RoadRunner?
**Ответ:** RoadRunner выступает потребителем и менеджером задач, получая задачи от различных брокеров и передавая их PHP-воркерам. Это централизует управление задачами без необходимости отдельных CLI-процессов. Поддерживаемые драйверы/брокеры:
- `amqp`, `beanstalk`, `redis`, `sqs`, `nats`, `boltdb` и `memory`.
![Очереди](https://docs.roadrunner.dev/images/jobs-scheme.png)

#### Как RoadRunner обрабатывает HTTP-запросы и заголовки?
**Ответ:** HTTP-плагин RoadRunner действует как быстрый веб-сервер, преобразующий входящие запросы в PSR-7 совместимые объекты для PHP-воркеров. Он поддерживает Gzip-сжатие, пользовательские заголовки и middleware на уровне Go. Также поддерживает потоковую отдачу ответов, позволяя PHP отправлять большие данные клиенту инкрементально.

### Senior
#### Что такое плагин Centrifuge в RoadRunner?
**Ответ:** Плагин Centrifuge интегрирует RoadRunner с Centrifugo или его Go-аналогом, обеспечивая обмен сообщениями в реальном времени (WebSockets, SockJS) в PHP-приложениях. RoadRunner обрабатывает тысячи постоянных соединений на Go, тогда как PHP-воркеры обрабатывают только бизнес-логику и события.

#### Каково назначение плагина Service?
**Ответ:** Плагин Service позволяет RoadRunner запускать и мониторить любой бинарник или скрипт как подпроцесс (sidecar). Он гарантирует, что процесс всегда запущен, автоматически перезапуская его при сбое. Идеален для фоновых задач: консьюмеры, экспортёры метрик или прокси-серверы.

#### Как работает плагин Locks?
**Ответ:** Плагин Locks предоставляет механизм распределённых блокировок. Он позволяет PHP-воркерам синхронизировать доступ к разделяемым ресурсам между несколькими воркерами или даже несколькими экземплярами RoadRunner, используя бэкенды Redis или локальную память.

#### Каковы лучшие практики для запуска RoadRunner в продакшене?
**Ответ:**
- **Управление процессами:** Используйте `systemd` или `supervisord` для поддержания основного процесса RoadRunner в работе.
- **Лимиты воркеров:** Всегда устанавливайте `max_jobs` или `max_memory` для предотвращения раздувания памяти.
- **Мониторинг здоровья:** Используйте плагин `status` для проверок работоспособности и готовности.
- **Настройка ОС:** Увеличьте количество разрешённых файловых дескрипторов (`ulimit -n`).
- **Логи:** Настройте структурированное логирование (JSON) для удобного импорта в системы управления логами.

#### Что такое Response Streaming и Health Checks в RoadRunner?
**Ответ:**
- **Response Streaming:** Позволяет PHP передавать данные клиенту инкрементально, снижая потребление памяти при загрузке больших файлов или потоковой передаче данных в реальном времени.
- **Health Checks:** Плагин `status` предоставляет эндпоинты `/health` (жив ли сервер?) и `/ready` (есть ли хотя бы один свободный воркер?). Они необходимы для Kubernetes liveness/readiness probes или проверок здоровья балансировщика нагрузки.
#### Как RoadRunner интегрируется с Temporal?
**Ответ:** RoadRunner может служить воркером для Temporal — движка оркестрации для сложных, stateful и долгоживущих рабочих процессов. PHP-разработчики могут писать логику workflow и activity на PHP, тогда как RoadRunner обрабатывает коммуникацию с сервером Temporal через протокол Goridge.

## 22. Стандарты PSR и PER

### Junior
#### Какой документ PSR описывает базовые стандарты кодирования?
**Ответ: PSR-1**
PSR-1 направлен на обеспечение высокой степени технической совместимости между общим PHP-кодом. Он охватывает базовые элементы кодирования: файлы, пространства имён, классы и методы.
- Используйте только теги `<?php` и `<?=`.
- Используйте только UTF-8 без BOM для PHP-кода.
- Файлы должны либо объявлять символы (классы, функции, константы и т. д.), либо вызывать побочные эффекты (генерировать вывод, менять настройки .ini и т. д.), но не то и другое одновременно.
- Пространства имён и классы должны следовать PSR-4.
- Имена классов должны объявляться в `StudlyCaps`.
- Константы классов должны объявляться полностью в верхнем регистре с разделителями-подчёркиваниями.
- Имена методов должны объявляться в `camelCase`.

#### Какой документ PSR описывает расширенные стандарты кодирования?
**Ответ: PSR-12**
PSR-12 — расширение PSR-2 (который он заменил), предоставляющее более полный набор правил стиля кодирования. **Примечание:** Он заменён «живым» стандартом [PER Coding Style](#21-стандарты-psr-и-per).
- Отступы должны быть 4 пробела, без табуляций.
- Мягкий лимит длины строки — 80 символов, жёсткий — 120.
- Фигурные скобки для классов и методов должны быть на новой строке.
- Видимость должна быть объявлена для всех свойств и методов.
- Type hints обязательны для параметров и возвращаемых значений, где это возможно.

#### Какой документ PSR описывает автозагрузку?
**Ответ: PSR-4**
PSR-4 описывает спецификацию автозагрузки классов из путей файлов. Это современная замена PSR-0.
- Он сопоставляет пространства имён с директориями.
- Позволяет использовать «префикс» (базовую директорию) для пространства имён.
- **Примечание о PSR-0:** PSR-0 был предыдущим стандартом, но устарел, так как был менее гибким (например, требовал полное отражение пространства имён в структуре каталогов, а подчёркивания в именах классов трактовались как разделители каталогов).

### Middle
#### Какой документ PSR описывает интерфейс логирования?
**Ответ: PSR-3**
PSR-3 предоставляет общий интерфейс для библиотек логирования. Он определяет `LoggerInterface` с методами для 8 уровней логирования (debug, info, notice, warning, error, critical, alert, emergency). Это позволяет приложениям быть независимыми от конкретной реализации логирования (например, Monolog).

#### Какой документ PSR описывает интерфейс HTTP-сообщений?
**Ответ: PSR-7**
PSR-7 описывает общие интерфейсы для представления HTTP-сообщений (запросов и ответов).
- HTTP-сообщения трактуются как неизменяемые (immutable) объекты.
- Охватывает запросы, ответы, загруженные файлы, URI и потоки.
- Обеспечивает интероперабельность между различными HTTP-библиотеками и фреймворками.

#### Какой документ PSR описывает HTTP-обработчики?
**Ответ: PSR-15**
PSR-15 определяет интерфейсы для серверных HTTP-компонентов: Request Handlers и Middleware.
- Middleware позволяет обрабатывать входящие запросы и исходящие ответы послойно.
- Работает в связке с HTTP-сообщениями PSR-7, предоставляя стандартный способ обработки веб-запросов.

### Senior
#### Какой документ PSR описывает интерфейс контейнера?
**Ответ: PSR-11**
PSR-11 предоставляет общий интерфейс для контейнеров внедрения зависимостей.

#### Какой документ PSR описывает Event Dispatcher?
**Ответ: PSR-14**
PSR-14 определяет общий интерфейс для диспетчеризации событий и подписки на них.
- Состоит из Event Dispatcher, Listener Provider и объекта Event.
- Предоставляет стандартный способ реализации точек расширения в PHP-приложениях и библиотеках.

#### Какой документ PSR описывает HTTP Factory?
**Ответ: PSR-17**
PSR-17 определяет интерфейсы для фабрик, создающих PSR-7 совместимые HTTP-объекты.
- Поскольку объекты PSR-7 неизменяемы, фабрики предоставляют стандартный способ создания новых экземпляров (Requests, Responses, Streams и т. д.) без привязки к конкретной реализации.

#### [ВАЖНО] Что такое PER Coding Style и почему он важен?
**Ответ: PER Coding Style**
**PHP Evolved Recommendation (PER)** для стиля кодирования — это современный преемник PSR-12. В отличие от PSR, которые статичны после принятия, PER — это «живые документы», которые могут обновляться с учётом новых возможностей PHP и развивающихся лучших практик.
- **Преемник PSR-12:** Он расширяет и заменяет PSR-12, по-прежнему требуя соблюдения PSR-1 (базовые стандарты кодирования).
- **Живой стандарт:** Обновляется для включения правил для нового синтаксиса PHP, такого как Constructor Property Promotion, Enums, Readonly Properties и других.
- **Цель:** Предоставить полный, развивающийся набор правил стиля кодирования, идущий в ногу с развитием языка, обеспечивая единообразие кода во всей экосистеме PHP.

## 23. Базовые алгоритмы

### Junior

#### Как вы перевернете строку в PHP?
**Ответ:** Перевернуть строку можно несколькими способами, от использования встроенных функций до ручных реализаций, учитывающих многобайтовые символы. [Подробный гайд](answers/reverse_string.md)

- **Junior:** Используйте встроенную функцию `strrev()`.
- **Middle:** Используйте ручной цикл `for`, который показывает понимание байтовой структуры строки, но не является безопасным для многобайтовых строк.
- **Senior:** Используйте подход, безопасный для многобайтовых строк (например, `mb_str_split()`), чтобы корректно обрабатывать символы UTF-8.

```php
// Junior
echo strrev("hello"); // "olleh"

// Senior (безопасно для UTF-8)
function mb_strrev(string $string, string $encoding = null): string {
    $chars = mb_str_split($string, 1, $encoding ?: mb_internal_encoding());
    return implode('', array_reverse($chars));
}
```

#### Что такое линейный поиск и как он реализуется в PHP?
**Ответ:** Линейный поиск — простейший алгоритм поиска. Он последовательно проверяет каждый элемент списка, пока не найдёт целевое значение или не достигнет конца списка.
- **Сложность:** O(n).
- **Применение:** Небольшие наборы данных или несортированные списки.

```php
function linearSearch(array $arr, $target): int
{
    foreach ($arr as $index => $value) {
        if ($value === $target) {
            return $index;
        }
    }
    return -1;
}
```

#### Что такое бинарный поиск и каковы его требования?
**Ответ:** Бинарный поиск — быстрый алгоритм поиска, который работает путём многократного деления интервала поиска пополам.
- **Требование:** Массив **должен быть отсортирован**.
- **Сложность:** O(log n).

```php
function binarySearch(array $arr, $target): int
{
    $low = 0;
    $high = count($arr) - 1;

    while ($low <= $high) {
        $mid = floor(($low + $high) / 2);
        if ($arr[$mid] === $target) {
            return $mid;
        }
        if ($arr[$mid] < $target) {
            $low = $mid + 1;
        } else {
            $high = $mid - 1;
        }
    }
    return -1;
}
```

#### Что такое пузырьковая сортировка (Bubble Sort)?
**Ответ:** Пузырьковая сортировка — простой алгоритм сортировки, который многократно проходит по списку, сравнивает соседние элементы и меняет их местами, если они расположены в неправильном порядке. Процесс повторяется, пока список не будет отсортирован.
- **Сложность:** O(n²).
- **Применение:** Учебные цели или очень маленькие наборы данных.

```php
function bubbleSort(array $arr): array
{
    $n = count($arr);
    for ($i = 0; $i < $n; $i++) {
        for ($j = 0; $j < $n - $i - 1; $j++) {
            if ($arr[$j] > $arr[$j + 1]) {
                // Обмен
                $temp = $arr[$j];
                $arr[$j] = $arr[$j + 1];
                $arr[$j + 1] = $temp;
            }
        }
    }
    return $arr;
}
```

### Middle

#### Как работает быстрая сортировка (Quick Sort)?
**Ответ:** Quick Sort — алгоритм «разделяй и властвуй». Он выбирает элемент в качестве «опорного» (pivot) и разбивает массив вокруг него так, чтобы элементы меньше опорного были слева, а большие — справа. Затем рекурсивно сортирует подмассивы.
- **Сложность:** O(n log n) в среднем, O(n²) в худшем случае.

```php
function quickSort(array $arr): array
{
    if (count($arr) < 2) {
        return $arr;
    }

    $pivot = $arr[0];
    $left = $right = [];

    for ($i = 1; $i < count($arr); $i++) {
        if ($arr[$i] < $pivot) {
            $left[] = $arr[$i];
        } else {
            $right[] = $arr[$i];
        }
    }

    return array_merge(quickSort($left), [$pivot], quickSort($right));
}
```

#### Что такое сортировка слиянием (Merge Sort) и в чём её главное преимущество?
**Ответ:** Merge Sort — ещё один алгоритм «разделяй и властвуй». Он делит массив на две половины, рекурсивно сортирует их и затем объединяет отсортированные половины.
- **Преимущество:** Гарантированная сложность O(n log n), даже в худшем случае. Также является «стабильной» сортировкой.

```php
function mergeSort(array $arr): array
{
    $count = count($arr);
    if ($count <= 1) {
        return $arr;
    }

    $mid = (int)($count / 2);
    $left = array_slice($arr, 0, $mid);
    $right = array_slice($arr, $mid);

    return merge(mergeSort($left), mergeSort($right));
}

function merge(array $left, array $right): array
{
    $result = [];
    while (count($left) > 0 && count($right) > 0) {
        if ($left[0] <= $right[0]) {
            $result[] = array_shift($left);
        } else {
            $result[] = array_shift($right);
        }
    }
    return array_merge($result, $left, $right);
}
```

#### Где следует применять эти алгоритмы в реальной PHP-разработке?
**Ответ:**
1. **Оптимизация поиска:** Используйте **бинарный поиск**, когда у вас есть большой, статичный, отсортированный набор данных (например, поиск по большому отсортированному списку ID товаров или названий).
2. **Управление ресурсами:** **Очереди** (FIFO) используются для обработки фоновых задач (например, Laravel Queues, RabbitMQ) для выполнения задач вроде отправки писем без блокировки основного запроса.
3. **Парсинг:** **Стеки** (LIFO) или рекурсивные алгоритмы используются для парсинга структурированных данных вроде HTML, XML или пользовательских шаблонизаторов для проверки корректности вложенности тегов/скобок.
4. **Отображение данных:** Алгоритмы **сортировки** необходимы для отображения данных в определённом порядке (по алфавиту, по цене, по дате).
