# Сравнение MySQL и PostgreSQL

На этой странице приведено сравнение MySQL и PostgreSQL, двух самых популярных реляционных систем управления базами данных (СУБД) с открытым исходным кодом.

## Обзор

| Характеристика | MySQL | PostgreSQL |
| :--- | :--- | :--- |
| **Тип** | Реляционная (RDBMS) | Объектно-реляционная (ORDBMS) |
| **Лицензия** | Двойная (GPL/Коммерческая) | Лицензия PostgreSQL (похожа на MIT/BSD) |
| **Соответствие SQL** | Частичное (ориентировано на простоту) | Высокое (строгое соблюдение стандартов) |
| **ACID-соответствие** | Да (с InnoDB) | Да (полностью ACID-совместима) |
| **Параллелизм** | Хороший (MVCC) | Отличный (продвинутый MVCC) |
| **Расширяемость** | Ограниченная | Высокая (кастомные типы, функции, операторы) |

## Ключевые различия

### 1. Типы данных
- **MySQL:** Стандартные типы (INT, VARCHAR, TEXT, DATETIME и т. д.). Ограниченная поддержка сложных типов.
- **PostgreSQL:** Богатый набор встроенных типов (геометрические, сетевые адреса, JSONB, массивы, диапазоны, перечисления). Поддерживает пользовательские типы.

### 2. Производительность и параллелизм
- **MySQL:** Традиционно известна высокой скоростью выполнения операций чтения. Использует блокировку на уровне строк в InnoDB.
- **PostgreSQL:** Превосходит в сложных запросах, операциях записи и при высокой конкуренции. Более широко использует MVCC (Multi-Version Concurrency Control).

### 3. Возможности
- **MySQL:** Фокусируется на скорости и простоте. Такие функции, как `JOIN` и `INDEX`, хорошо оптимизированы.
- **PostgreSQL:** Продвинутые функции, такие как оконные функции, общие табличные выражения (CTE), полнотекстовый поиск и продвинутое индексирование (GIN, GiST, BRIN).

### 4. Поддержка JSON
- **MySQL:** Поддерживает тип данных JSON с функциональными индексами.
- **PostgreSQL:** Превосходная поддержка JSON с использованием `JSONB` (бинарный формат), что позволяет эффективно индексировать и запрашивать вложенные данные.

### 5. Репликация
- **MySQL:** Сильная встроенная репликация (Master-Slave, Master-Master).
- **PostgreSQL:** Поддерживает синхронную, асинхронную и логическую репликацию.

## Когда что выбирать?

### Выбирайте MySQL, если:
- Вам нужна простая, легкая в настройке база данных для стандартного веб-приложения.
- Ваша нагрузка в основном состоит из операций чтения.
- Вы используете CMS, такие как WordPress, Drupal или Joomla.

### Выбирайте PostgreSQL, если:
- У вас сложные структуры данных или нужны кастомные типы.
- Вам требуется строгая целостность данных и соответствие стандартам SQL.
- Ваше приложение включает сложные аналитические запросы (BI, Data Science).
- Вам нужны продвинутые функции, такие как полнотекстовый поиск или географические данные (PostGIS).

## Сводная таблица

| Функция | MySQL | PostgreSQL |
| :--- | :--- | :--- |
| **Соединения (Joins)** | Оптимизированы для простых соединений | Лучше для сложных многотабличных соединений |
| **Индексы** | B-Tree, Hash, Spatial, Full-text | B-Tree, Hash, GiST, GIN, BRIN, SP-GiST |
| **Хранимые процедуры** | Поддерживаются (на основе SQL) | Продвинутые (PL/pgSQL, Python, Perl и т. д.) |
| **Триггеры** | Базовая поддержка | Продвинутая поддержка (на строку, на выражение) |
