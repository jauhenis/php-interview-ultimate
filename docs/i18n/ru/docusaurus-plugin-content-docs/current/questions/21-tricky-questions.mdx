# 21. Каверзные вопросы

### В чём разница между `==` и `===`?

<span className="badge badge--success margin-bottom--md">Junior</span>

**Ответ:**

- `==`: Равенство (проверяет значение, выполняет приведение типов).
- `===`: Идентичность (проверяет и значение, и тип).

---

### Что произойдёт при сравнении "10" и 10 с помощью `==`?

<span className="badge badge--success margin-bottom--md">Junior</span>

**Ответ:** Вернётся `true` из-за приведения типов (type juggling).

---

### Что выведет следующий код?

<span className="badge badge--success margin-bottom--md">Junior</span>

```php
function foo(&$var) {
    $var++;
}
$a = 5;
foo($a);
echo $a;
```

**Ответ:** `6`. Оператор `&` передаёт переменную по ссылке, поэтому изменение внутри функции влияет на оригинальную переменную.

---

### В чём разница между `array_merge()` и оператором `+` для массивов?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:**

- `array_merge()`: Для строковых ключей последнее значение перезаписывает предыдущее. Для числовых ключей значения добавляются в конец и переиндексируются.
- `+`: Для строковых ключей сохраняется первое значение, второе игнорируется. Для числовых ключей также сохраняется первое.

---

### Как ведут себя статические свойства в наследуемых классах?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:** Статические свойства являются общими для всех классов в иерархии наследования, если только они не переопределены явно в дочернем классе.

---

### Почему использование PHP в качестве демона связано с рисками?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:** PHP изначально спроектирован для обработки кратковременных запросов. В долгоживущем процессе утечки памяти в вашем коде или библиотеках со временем приведут к аварийному завершению процесса.
