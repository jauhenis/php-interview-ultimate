# 15. Инфраструктура, Docker и DevOps



### Что такое Docker?

<span className="badge badge--success margin-bottom--md">Junior</span>

**Ответ:** Платформа для разработки, доставки и запуска приложений в контейнерах, гарантирующая одинаковую работу приложения в любом окружении.

---

### Docker ENV и ARG: в чём разница?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:**

- `ARG`: Переменные, доступные только во время сборки образа.
- `ENV`: Переменные, доступные как во время сборки, так и во время работы контейнера.

---

### Что такое Docker Volumes?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:** Способ сохранения данных, генерируемых и используемых контейнерами Docker, независимо от жизненного цикла контейнера.

---

### В чём разница между горизонтальным и вертикальным масштабированием?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:**

- **Вертикальное масштабирование:** Добавление ресурсов (CPU, RAM) к одному серверу.
- **Горизонтальное масштабирование:** Добавление новых серверов для распределения нагрузки.

---

### Что такое CI/CD?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:**

- **Непрерывная интеграция (CI):** Автоматизация сборки и тестирования кода при каждом коммите.
- **Непрерывная доставка/развёртывание (CD):** Автоматизация развёртывания кода на продакшен- или стейджинг-окружения.

---

### RabbitMQ и Kafka: сравнение

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:**

- **RabbitMQ:** Брокер сообщений, ориентированный на доставку сообщений потребителям (Push-модель). Отлично подходит для сложной маршрутизации.
- **Kafka:** Распределённая потоковая платформа, ориентированная на высокую пропускную способность и сохранение данных (Pull-модель). Отлично подходит для обработки логов.
