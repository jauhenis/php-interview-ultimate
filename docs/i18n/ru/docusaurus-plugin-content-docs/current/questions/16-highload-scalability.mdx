# 16. Высокие нагрузки и масштабируемость

### Что такое балансировка нагрузки (Load Balancing)?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:** Распределение сетевого или прикладного трафика между несколькими серверами для повышения отзывчивости и доступности.

---

### Как оптимизировать медленный GET-эндпоинт, возвращающий большое количество записей?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:**

- Использовать пагинацию.
- Использовать кеширование (Redis/Memcached).
- Оптимизировать запросы к базе данных (индексирование, выборка только нужных столбцов).
- Использовать жадную загрузку (Eager Loading) для предотвращения проблемы N+1.

---

### Что такое CAP-теорема?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:** В распределённой системе можно обеспечить только два из трёх свойств одновременно: согласованность (Consistency), доступность (Availability) и устойчивость к разделению сети (Partition Tolerance).

---

### Что такое репликация базы данных?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:** Копирование данных с одного сервера базы данных (Master) на другой (Slave) для повышения надёжности, отказоустойчивости и производительности.
