# 20. Чистый код и лучшие практики



### Что такое DRY и KISS?

<span className="badge badge--success margin-bottom--md">Junior</span>

**Ответ:**

- **DRY:** Don't Repeat Yourself — не повторяйся.
- **KISS:** Keep It Simple, Stupid — делай проще.

---

### Что такое YAGNI?

<span className="badge badge--success margin-bottom--md">Junior</span>

**Ответ:** You Ain't Gonna Need It — «Тебе это не понадобится». Не реализуйте функциональность, пока она действительно не потребуется.

---

### Что такое «Композиция вместо наследования» (Composition over Inheritance)?

<span className="badge badge--primary margin-bottom--md">Middle</span>

**Ответ:** Принцип, предлагающий достигать полиморфного поведения и повторного использования кода путём композиции объектов с другими объектами, реализующими нужную функциональность, вместо наследования от базового или родительского класса.

---

### Представьте, что у вас есть старый класс в монолите, содержащий множество методов, и его нужно изменить. Как бы вы провели рефакторинг?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:**

1. **Разобраться и покрыть тестами:** Написать тесты для существующей функциональности, чтобы гарантировать отсутствие регрессий при рефакторинге.
2. **Определить зоны ответственности:** Выявить ключевые обязанности класса и вынести их в отдельные классы с единственной ответственностью (SRP).
3. **Убрать связанность:** Использовать Dependency Injection для отвязки класса от его зависимостей, повышая тестируемость.
4. **Инкрементальный рефакторинг:** Проводить рефакторинг небольшими шагами, запуская тесты после каждого изменения.
5. **Применять паттерны проектирования:** Использовать паттерны Strategy, Factory или Service-объекты для обработки сложной логики или множественных условных ветвлений.
   [Подробное руководство по паттернам проектирования](../answers/design_patterns.md)

### Что такое закон Деметры (Law of Demeter)?

<span className="badge badge--warning margin-bottom--md">Senior</span>

**Ответ:** Принцип проектирования, согласно которому модуль не должен знать о внутреннем устройстве объектов, с которыми он работает. «Не разговаривай с незнакомцами.»
